# Fabric 선택 종료 및 커서 안정화 설계

- 상태: 사용자 승인 완료
- 작성일: 2026-07-21
- 기준 브랜치: `codex/baeframe-fabric-brush-style-v1`
- 기준 커밋: `20072d9`

## 1. 배경

Fabric 시험판에서 `V`로 두 개 이상의 획을 사각 선택해 이동하면 Fabric의 `ActiveSelection`이 이동 후에도 남는다. 이 객체는 선택된 획들을 감싸는 직사각형 전체를 입력 대상으로 사용하므로 다음 문제가 함께 발생한다.

1. 이동 직후 획을 다시 드래그해도 개별 획이 아니라 이전 다중 선택 전체가 잡힌다.
2. 획 사이의 빈 공간에서도 이동 커서가 표시된다.
3. 개별 획은 `perPixelTargetFind`를 사용하지만 선택 여유가 0이어서 얇은 획의 정확한 색칠 픽셀을 맞혀야만 hover와 선택이 활성화된다.

실제 Fabric 7.4.0으로 재현한 결과, 빈 곳을 한 번 클릭해 `ActiveSelection`을 해제하면 개별 획 이동은 다시 정상 작동했다. 따라서 데이터 손상이나 좌표 저장 실패가 아니라 선택 상태 종료가 빠진 문제다.

## 2. 목표

- 다중 선택 이동을 놓은 뒤 다음 획을 별도의 빈 곳 클릭 없이 바로 선택하고 이동할 수 있어야 한다.
- 얇은 획도 화면에서 무리 없이 가리킬 수 있어야 한다.
- 빈 곳, 선택 가능한 획, 이동 중 상태를 커서가 정확히 구분해야 한다.
- 선택 해제 자체가 장면 변경이나 저장 시도로 기록되면 안 된다.
- 기존의 B 토글, 재생, 타임라인, 단일 획 이동, 이동 전용 제약을 유지해야 한다.

## 3. 선택한 UX

다중 선택 이동은 한 번의 이동으로 확정되는 방식으로 처리한다.

1. 사용자가 `V`로 여러 획을 사각 선택한다.
2. 선택 묶음을 드래그해 놓는다.
3. 선택된 모든 획의 이동값을 장면 저장소에 정확히 한 번 반영한다.
4. Fabric의 현재 이동 이벤트가 완전히 끝난 다음 선택 상자만 자동 해제한다.
5. `V` 도구는 유지한다.
6. 다음 hover와 드래그는 즉시 개별 획을 대상으로 한다.

사각 선택만 하고 아직 이동하지 않은 상태에서는 선택을 유지한다. 단일 획 이동 후에도 해당 획 선택은 유지한다. 자동 해제 대상은 실제 이동이 반영된 다중 선택뿐이다.

## 4. 입력 및 커서 계약

| 상태 | 커서 | 입력 대상 |
| --- | --- | --- |
| 브러시 모드 | `crosshair` | 새 획 그리기 |
| `V` 모드의 빈 곳 | `default` | 사각 선택 시작 |
| `V` 모드의 획 위 | `grab` | 개별 획 선택 또는 이동 |
| 선택 항목 이동 중 | `grabbing` | 현재 선택 이동 |
| 드로잉 입력 비활성 | 드로잉 커서 없음 | 아래의 기존 배프레임 UI |

선택 여유는 화면 기준 6px를 목표로 한다. Fabric은 대략적인 경계 판정과 픽셀 판정을 연속으로 수행하므로 다음 두 값을 함께 적용한다.

- 각 영구 Path의 `padding`
- Canvas의 `setTargetFindTolerance(...)`

Fabric의 내부 캔버스는 원본 영상 해상도이고 화면에는 CSS 크기와 줌으로 표시된다. 따라서 6px을 고정 원본 좌표로 넣지 않고 다음 규칙으로 Fabric 논리 좌표에 환산한다.

1. `sourceWidth / canvasRect.width / viewportScale`과 `sourceHeight / canvasRect.height / viewportScale`을 구한다.
2. 두 값 중 큰 값에 6을 곱하고 올림한다.
3. 결과를 2~96 범위의 정수로 제한한다.

같은 환산값을 Path `padding`과 Canvas tolerance에 사용한다. viewport가 바뀌면 기존 획과 Canvas tolerance를 함께 갱신한다.

## 5. 상태 전환 구현

### 5.1 다중 선택 이동 확정

기존 `object:modified` 경로가 다중 선택의 `dx`, `dy`를 장면 저장소에 반영하는 책임을 계속 가진다. 적용 결과가 실제 변경이고 대상이 두 개 이상의 획을 가진 `ActiveSelection`일 때만 후속 선택 해제를 예약한다.

`object:modified` 처리 중에 동기적으로 `discardActiveObject()`를 호출하면 Fabric이 아직 현재 transform을 종료하지 않은 상태라 같은 수정 이벤트가 재진입할 수 있다. 따라서 해제는 microtask로 미룬다.

예약된 작업은 다음 조건을 모두 다시 확인한다.

- 런타임이 파괴되지 않았고 입력이 활성 상태다.
- 같은 세션과 입력 revision이 계속 유효하다.
- 예약 당시의 `ActiveSelection`이 아직 현재 active object다.

조건이 맞을 때만 active object를 해제하고 다시 그린다. `selection:cleared`는 장면 저장소의 선택 ID만 비우며 mutation, dirty, save attempt를 증가시키지 않는다. 다른 도구 전환, B 해제, 프레임 변경이 먼저 발생했다면 예약 작업은 아무 일도 하지 않는다.

### 5.2 커서 소유권

커서 정책은 Fabric 기본값에 맡기지 않고 overlay runtime이 명시적으로 설정한다.

- Canvas: 도구별 기본 커서
- 영구 Path: `hoverCursor: 'grab'`, `moveCursor: 'grabbing'`
- ActiveSelection: 선택 생성 시 같은 이동 커서와 이동 전용 제약 적용
- 다중 선택 자동 해제 및 입력 비활성 시 남아 있는 이동 커서 초기화

브러시 미리보기처럼 transient인 Path는 선택·hover 대상이 아니다.

## 6. 테스트 설계

### 6.1 실제 Fabric 회귀 테스트

가짜 Canvas에 선택 배열을 직접 주입하는 기존 테스트만으로는 `ActiveSelection`의 hit-test와 이벤트 종료 순서를 검증할 수 없다. 설치된 `fabric/node`를 사용하는 실제 Fabric 테스트를 추가한다.

시나리오:

1. 획 두 개 생성
2. `V` 전환
3. 빈 영역에서 사각 선택해 두 획 선택
4. 다중 선택을 이동
5. 이벤트 settle 후 active selection과 저장소 selection이 모두 비었는지 확인
6. 빈 곳 클릭 없이 첫 번째 획을 즉시 드래그
7. 첫 번째 획만 이동하고 두 번째 획은 그대로인지 확인
8. mutation은 획 생성 2회, 다중 이동 1회, 단일 이동 1회의 총 4회인지 확인
9. save attempt가 0인지 확인

### 6.2 hit-test 및 커서 테스트

- 얇은 획의 가장자리에서 화면 기준 약 6px 안은 획 대상으로 판정한다.
- 허용 범위를 벗어난 빈 곳은 대상으로 판정하지 않는다.
- 떨어진 두 획 사이 빈 공간은 `default`다.
- 획 위는 `grab`, 실제 이동 중은 `grabbing`, 브러시는 `crosshair`다.
- 50%, 100%, 200% viewport scale과 서로 다른 원본/표시 크기에서 선택 여유가 과도하게 달라지지 않는다.

### 6.3 기존 회귀검증

- 이번 기능 집중 테스트
- `npm run test:drawing`
- `npm run test:mpv`
- overlay 번들 재생성 후 git diff 확인
- unpacked 앱 빌드 및 app.asar 번들 일치 확인
- 고유 profile과 `--skip-shell-registration`을 사용한 격리 실행

## 7. 수동 합격 기준

1. 재생 중 `B`로 진입해 두 획을 그린다.
2. `V`로 두 획을 사각 선택해 이동한다.
3. 선택 상자가 자동으로 사라지되 `V`는 유지된다.
4. 즉시 한 획을 가리키면 `grab`이 표시된다.
5. 그 획을 드래그하면 그 획만 움직인다.
6. 얇은 획을 정확히 한 픽셀로 맞히지 않아도 안정적으로 선택된다.
7. 빈 공간에서는 이동 커서가 나오지 않는다.
8. 이후 B와 Space가 계속 작동하며 깜빡임, 이중 재생, 타임라인 점프가 없다.
9. 리뷰 데이터 저장 실패 알림이 없다.

## 8. 이번 범위에서 제외

- 라쏘나 사각형으로 획의 일부를 잘라내는 기능
- 다중 선택을 유지한 채 내부 획을 직접 재선택하는 고급 sub-target 동작
- Undo/Redo
- 지우개, 도형, 레이어 저장, 앱 재시작 후 영속화
- 기존 HTML 드로잉 엔진 변경
