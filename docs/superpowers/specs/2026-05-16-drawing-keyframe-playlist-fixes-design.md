# BAEFRAME 드로잉/키프레임/재생목록 자잘한 개선 설계

작성일: 2026-05-16

## 1. 목적

이번 작업은 큰 화면 개편이 아니라 실제 작업 중 걸리는 작은 불편을 줄이는 데 집중한다.

- 지우개에 `픽셀 지우기`와 `획 지우기`를 추가한다.
- 펜/브러시 도구에서 `Ctrl`을 누른 채 그리면 임시 지우개로 동작하게 한다.
- 액팅 타임라인에서 여러 키프레임을 드래그로 선택하고 함께 이동/삭제할 수 있게 한다.
- 재생목록 자동재생 토글이 켜져 있으면 별도 `전체 자동재생` 버튼 없이도 다음 영상으로 넘어가 재생되게 한다.
- `.bplaylist` 파일을 일반 열기/드롭/재생목록 추가 흐름에서 자연스럽게 열 수 있게 한다.
- 재생목록 연속 재생의 컷 전환 지연을 줄인다.

## 2. 드로잉 지우개

지우개 모드는 두 가지다.

- `픽셀 지우기`: 기존 지우개 동작이다. 기본값으로 유지한다.
- `획 지우기`: 앞으로 새로 그리는 펜/브러시 자유선 기록만 대상으로 한다.

기존 그림은 한 장의 이미지처럼 저장되어 있어 어떤 선이 하나의 획이었는지 알 수 없다. 그래서 기존 작업을 마이그레이션하지 않는다. 기존 그림이나 획 기록이 없는 선은 `획 지우기`에서 지워지지 않는다.

UI는 기존 드로잉 도구 패널 안에 둔다. 지우개를 선택했을 때 크기/불투명도 같은 설정을 조정하는 영역에 `픽셀 지우기` / `획 지우기` 선택 버튼을 표시한다. 마지막 선택값은 이 PC의 사용자 설정에 저장한다.

## 3. 획 기록 저장

새로 그리는 펜/브러시 자유선은 키프레임에 `strokeRecords`로 저장한다. 저장 후 다시 열어도 이 기록은 남아야 한다.

각 키프레임은 기존 `canvasData`를 계속 가진다. 호환성과 빠른 렌더링을 위해 완성 이미지도 저장해야 하기 때문이다. 새로 추가되는 정보는 다음 두 가지다.

- `baseCanvasData`: 획 기록이 쌓이기 전 바탕 이미지
- `strokeRecords`: 이후에 개별 삭제 가능한 자유선 기록

`획 지우기`로 획을 삭제하면 `baseCanvasData` 위에 남은 `strokeRecords`를 다시 그려 `canvasData`를 갱신한다.

`픽셀 지우기`, 도형 그리기, 전체 지우기처럼 획 단위로 되돌릴 수 없는 작업을 하면 그 시점의 결과를 이미지로 굳히고 기존 `strokeRecords`는 비운다. 이후 새로 그린 펜/브러시 자유선부터 다시 획 지우기 대상이 된다.

## 4. 획 지우기 동작

대상은 현재 선택된 레이어의 현재 프레임에 보이는 실제 키프레임이다.

애니메이트 방식의 홀드 프레임을 따른다. 예를 들어 1프레임에 키프레임과 그림이 있고 3프레임에서 그 그림이 이어져 보이는 상태라면, 3프레임에서 지워도 새 키프레임을 만들지 않고 1프레임의 원본 키프레임을 수정한다.

지우개 범위는 현재 지우개 크기를 그대로 쓴다. 지우개 범위에 여러 획이 닿으면 닿은 획을 모두 삭제한다. 펜/브러시 자유선만 대상이고, 선/화살표/사각형/원 도형은 이번 범위에서 제외한다.

펜/브러시 도구에서 `Ctrl`을 누른 채 획을 시작하면 해당 획 전체가 임시 지우개로 동작한다. 시작 시점의 `Ctrl` 상태를 기준으로 고정하며, 그리는 중에 `Ctrl`을 누르거나 떼도 현재 획의 성격은 바뀌지 않는다.

## 5. 키프레임 다중 조작

현재 타임라인에는 선택/드래그 구조가 일부 있지만, 실제 렌더링되는 키프레임 점과 선택 박스 대상이 어긋나 있다. 이 부분을 고쳐 사용자가 드래그 박스로 여러 키프레임을 선택할 수 있게 한다.

확정 동작은 다음과 같다.

- 드로잉 레이어 트랙 빈 공간에서 드래그하면 선택 박스가 생긴다.
- 선택 박스 안에 들어온 키프레임 점이 선택된다.
- `Ctrl`을 누른 상태로 드래그 선택하면 기존 선택에 추가한다.
- 선택된 키프레임 중 하나를 드래그하면 선택된 키프레임이 같은 프레임 수만큼 함께 이동한다.
- `Delete`를 누르면 선택된 키프레임을 한 번에 삭제한다.
- 선택된 키프레임이 없으면 기존처럼 현재 프레임의 키프레임 삭제 동작을 유지한다.

## 6. 재생목록 자동재생과 파일 열기

기존 `.bplaylist` 파일 형식은 유지한다.

일반 파일 열기, 드롭, 재생목록 추가 영역에서 `.bplaylist`를 받으면 재생목록 파일로 열고 사이드바를 표시한다. 재생목록 추가 버튼의 파일 선택창도 `.bplaylist`를 선택할 수 있어야 한다.

개별영상 모드에서 `자동 재생` 토글이 켜져 있고 현재 영상이 끝나면 다음 영상으로 이동한 뒤 자동으로 재생한다. 더 이상 `전체 자동재생` 버튼을 눌러야만 자동 재생되는 느낌이 들면 안 된다.

`전체 자동재생` 버튼은 제거한다. 스페이스바나 재생 버튼을 눌렀을 때 `자동 재생` 토글이 켜져 있고 타임라인 이어붙이기 모드라면 이어붙이기 재생을 바로 시작한다.

`자동 재생` 토글은 `.bplaylist` 파일에 저장하지 않는다. 테마 설정처럼 이 PC의 사용자 설정으로만 다루며, 재생목록 파일을 공유해도 다른 사람의 자동재생 상태를 바꾸지 않는다.

## 7. 컷 전환 딜레이 완화

완전히 한 영상처럼 무지연으로 붙이려면 다음 영상을 별도 플레이어에 미리 로드하거나 mpv 플레이리스트 레벨로 넘기는 큰 구조 변경이 필요하다. 이번 작업에서는 기존 구조 안에서 체감 지연을 줄인다.

- 다음 영상 준비/캐시 확인은 현재처럼 미리 시작한다.
- 시작 전에 재생목록 전체를 순서대로 검사하지 않고, 현재 재생할 영상만 먼저 확인한다.
- 나머지 영상의 파일 확인은 재생을 시작한 뒤 백그라운드에서 처리한다.
- 다음 컷으로 넘어간 뒤 재생 가능 이벤트를 오래 기다리기보다 즉시 `play()`를 시도한다.
- 재생이 실제로 진행되는지만 짧게 감시하고, 실패하면 기존처럼 재시도/건너뛰기 처리한다.

이 방식은 저장 구조나 플레이어 구조를 갈아엎지 않고 컷 전환 대기 시간을 줄이는 1차 개선이다.

## 8. 테스트 기준

실사용자 기준:

- 지우개 기본값이 픽셀 지우기로 시작한다.
- 지우개 도구 설정 영역에서 픽셀/획 지우기를 바꿀 수 있다.
- 앱을 다시 켜도 마지막 지우개 모드가 유지된다.
- 펜/브러시로 새로 그린 선은 `획 지우기`로 전체 삭제된다.
- 기존 그림이나 픽셀로 굳힌 그림은 `획 지우기`로 지워지지 않는다.
- 펜/브러시에서 `Ctrl`을 누르고 시작하면 현재 지우개 모드대로 임시 지우기가 된다.
- 여러 키프레임을 드래그로 선택하고 함께 이동/삭제할 수 있다.
- 자동 넘어가기가 켜져 있으면 영상 종료 후 다음 영상이 자동으로 재생된다.
- `.bplaylist`를 열거나 재생목록 추가 흐름에서 선택하면 재생목록이 열린다.

개발자 기준:

- 기존 `.bframe` 파일은 `strokeRecords`가 없어도 정상 로드된다.
- 새 `.bframe` 파일은 `strokeRecords`와 `baseCanvasData`를 저장하고 다시 복원한다.
- 픽셀 지우기 후에는 이전 획 기록이 굳혀져 더 이상 획 지우기 대상이 되지 않는다.
- 선택 박스는 실제 렌더링되는 `.keyframe-marker-dot`을 대상으로 한다.
- 재생목록 테스트는 기존 이어붙이기/정렬 테스트를 깨지 않아야 한다.
