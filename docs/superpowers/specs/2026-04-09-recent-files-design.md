- **작성일**: 2026-04-09
- **작성자**: baehandoridori (브레인스토밍), Claude (문서화)
- **대상 제품**: BAEFRAME (Electron 기반 비디오 리뷰 도구)
- **상태**: 설계 승인 완료, 구현 계획 작성 대기

---

## 1. 목적

배프레임을 쓰는 스튜디오 팀원이 **최근에 작업하던 영상 파일을 빠르게 다시 열 수 있도록** 한다. 현재는 매번 파일 다이얼로그에서 Google Drive의 깊은 경로를 다시 파고들어야 하므로, 프로젝트 간 빠른 전환과 앱 재시작 후 작업 이어가기가 번거롭다.

### 해결하려는 구체적 불편함 (사용자 확인)

1. 먼 경로 재탐색이 귀찮음 — 한 번에 "어제 보던 그 파일"로 점프하고 싶다
2. 여러 프로젝트/컷 사이를 빠르게 스위칭하고 싶다
3. 앱 재시작 후 직전 작업 컨텍스트를 빠르게 되찾고 싶다

### 하지 않을 것 (YAGNI)

- **자동 로드**: 앱 시작 시 마지막 파일을 자동으로 열지 않는다. 사용자가 항상 선택 단계를 거친다.
- **검색/필터**: 목록이 최대 15개라 검색이 불필요하다.
- **팀 공유**: 최근 목록은 각자 로컬에만 저장. Google Drive나 `.bframe`에 포함되지 않는다.
- **콘텐츠 해시 기반 동일 파일 추적**: 경로 기반 ID만 사용. 파일이 이동하면 새 항목으로 취급.

## 2. 사용자 시나리오

### 시나리오 A — 아침에 앱을 켠다
1. 앱 시작 → 빈 드롭존 화면이 뜬다
2. 드롭존 하단에 "최근 파일" 섹션이 있고, 어제 마지막으로 본 컷이 썸네일과 함께 맨 위에 있다
3. 클릭 한 번으로 열림 → 어제 작업 이어가기

### 시나리오 B — 프로젝트 간 빠른 전환
1. EP03 컷을 보던 중 EP02를 확인해야 함
2. 헤더 좌측 시계 아이콘 클릭 → 드롭다운 패널이 열림
3. 최근 목록에서 EP02 컷을 찾아 클릭 → 전환
4. 다시 EP03으로 돌아오려면 같은 방식

### 시나리오 C — 자주 쓰는 파일 고정
1. "오프닝 시퀀스" 같은 자주 참조하는 컷에 호버
2. 우측에 나타난 북마크 아이콘 클릭 → 고정됨
3. 고정된 항목은 목록 상단 "고정" 섹션에 상시 노출, 오래돼도 밀려나지 않음
4. 최대 5개까지 고정 가능

### 시나리오 D — 파일이 없어진 경우
1. 최근 목록의 항목이 회색으로 표시되고 우상단에 경고 아이콘
2. 클릭하면 "파일을 찾을 수 없습니다. 목록에서 제거할까요?" 다이얼로그
3. [제거] 선택 시 목록에서 사라짐

## 3. 기능 범위

### 3.1 기록 단위
**미디어 파일 단위**. `loadVideo(filePath)` 호출이 성공한 시점에 기록한다. 같은 영상이 여러 `.bframe` 리뷰 파일과 연결돼 있어도 한 항목으로 취급한다.

### 3.2 보관 정책
- 총 15개 (고정 5개 + 일반 10개)
- **고정(핀)**: 사용자가 명시적으로 고정한 항목. 최대 5개. 절대 자동 트림되지 않음.
- **일반**: 마지막 열람 시각 내림차순 정렬. 초과 시 가장 오래된 것부터 자동 제거.
- 4번째, 6번째 고정 시도 시 토스트 "고정은 최대 5개까지 가능합니다".

### 3.3 관리 기능
- 개별 항목 제거 (호버 시 우측 X 버튼, 또는 우클릭 메뉴)
- 전체 지우기 (빈 상태 화면 우측 하단 텍스트 버튼)
- 깨진 경로(없어진 파일) 자동 회색 처리 + 클릭 시 제거 확인
- 우클릭 컨텍스트 메뉴: 열기 / 파일 위치에서 보기 / 고정 토글 / 제거 / 경로 복사

### 3.4 썸네일
- 영상 길이의 **50% 지점**에서 대표 프레임 1장 캡처
- 크기 320×180, JPEG 품질 0.75 (~15KB)
- 오디오 파일은 썸네일 생략, 아이콘 폴백
- 캡처 실패 시 파일 종류별 아이콘 폴백 (영상: film 아이콘 / 오디오: music 아이콘)
- 한 번 성공하면 재캡처 없음 (파일 크기가 달라지면 재캡처)

## 4. UI 디자인

### 4.1 아이콘 체계
feather/lucide 스타일 인라인 SVG. 기존 `index.html`에서 사용하는 스타일과 동일:
```
viewBox="0 0 24 24" fill="none" stroke="currentColor"
stroke-width="2" stroke-linecap="round" stroke-linejoin="round"
```

| 용도 | 아이콘 이름 |
|------|------------|
| 최근 섹션/헤더 버튼 | `clock` (시계) |
| 고정 섹션/핀 버튼 | `bookmark` (활성 시 `fill="currentColor"`) |
| 누락 파일 경고 | `alert-triangle` |
| 항목 제거 | `x` |
| 드롭다운 화살표 | `chevron-down` (기존 version-dropdown에서 사용 중) |
| 비디오 썸네일 폴백 | `film` (기존 file-icon과 동일) |
| 오디오 썸네일 폴백 | `music` (기존 audio-mode에서 사용 중) |
| 폴더에서 보기 | `folder` |

**이모지 사용 금지**. 모든 시각 요소는 인라인 SVG로 통일하며 `currentColor`로 테마 색을 상속한다.

### 4.2 빈 상태 (드롭존 확장)
기존 드롭존(로고 + 안내 + "파일 열기" 버튼)은 그대로 유지하고 그 아래에 "최근 파일" 섹션을 이어 붙인다.

- 목록 0개일 때: 섹션 전체 숨김 (기존 드롭존 그대로)
- 고정 항목이 있으면 "고정" 소형 섹션 헤더 + 해당 항목들
- 그 아래 "최근" 섹션 헤더 + 일반 항목들
- 최대 8개까지 바로 표시, 초과 시 "더 보기" 버튼으로 나머지 전개
- 우측 하단에 "모두 지우기" 텍스트 버튼 (tertiary 스타일)
- 드래그앤드롭은 리스트 영역 위에서도 그대로 작동

### 4.3 헤더 버튼 (파일이 열린 상태)
헤더 좌측 로고와 파일정보 사이에 새 아이콘 버튼 `btnRecentFiles` 추가.
- 버튼 내용: `clock` + `chevron-down` 두 개 SVG 나란히 (텍스트 라벨 없음)
- 툴팁 "최근 파일"
- 스타일: 기존 `version-dropdown-trigger` 토큰 재사용
- 클릭 시 드롭다운 패널 토글

### 4.4 드롭다운 패널
`version-dropdown-menu`와 동일한 디자인 토큰으로 시각 일관성 확보. 클래스명은 `recent-dropdown-menu`.

구조:
- 헤더: `clock` 아이콘 + "최근 파일" 라벨
- 고정 섹션: `bookmark` 아이콘 + "고정" 라벨 + 카드들
- 분리선
- 일반 섹션: 카드들
- 분리선
- 푸터 액션: `folder` 아이콘 + "다른 파일 열기…" (기존 파일 다이얼로그 호출)

동작:
- 최대 높이 480px, 초과 시 내부 스크롤
- 바깥 클릭 / Esc 닫힘
- `↑/↓` 키보드 네비게이션, Enter 열기, Delete 제거

### 4.5 항목(카드) 구성
```
┌──┐  파일명.mp4                      10분 전    [bookmark]  [x]
│썸│  G:/…/경로 · 00:24                          (호버 시 노출)
└──┘
```
- 좌측: 80×45 썸네일 (16:9)
- 중앙: 파일명 (1줄 말줄임) / 경로 + 재생시간 (1줄 말줄임)
- 우측: 상대 시각 (10분 전, 어제, 2일 전, 지난주, …) + 호버 시 액션 버튼 2개
- 고정된 항목은 좌상단에 채워진 `bookmark` 배지

### 4.6 항목 인터랙션
| 동작 | 결과 |
|------|------|
| 좌클릭 | `loadVideo(item.path)` 호출. 파일 없으면 회색 처리된 상태에서만 확인 다이얼로그. |
| 호버 북마크 버튼 | 고정/해제 토글. 5개 초과 시 토스트 안내. |
| 호버 X 버튼 | 즉시 제거 + 3초간 "되돌리기" 토스트 |
| 우클릭 | Electron 네이티브 컨텍스트 메뉴 (열기 / 위치에서 보기 / 고정 / 제거 / 경로 복사) |

### 4.7 상태별 표시
| 상태 | 표시 |
|------|------|
| 목록 0개 | 섹션 전체 숨김 |
| 썸네일 생성 중 | placeholder 그라데이션 + shimmer |
| 썸네일 실패 (영상) | `film` 아이콘 placeholder |
| 썸네일 실패 (오디오) | `music` 아이콘 placeholder |
| 파일 누락 | opacity 0.5 + `alert-triangle` 우상단 오버레이 |
| 저장소 로드 실패 | 조용히 빈 배열 폴백, 사용자 차단 금지 |

### 4.8 접근성·키보드
- 모든 버튼에 `aria-label`
- 카드에 `role="listitem"`, 컨테이너에 `role="list"`
- Tab 순회, Enter 열기, Delete 제거, Shift+P 고정 토글

## 5. 데이터 모델

### 5.1 저장 위치
- **메타데이터**: `electron-store`를 사용해 `recent-files.json` 파일에 저장 (Electron 사용자 데이터 폴더)
- **썸네일 이미지**: `%APPDATA%/baeframe/recent-thumbs/{id}.jpg`
- **공유 범위**: 로컬 전용. 다른 팀원·Google Drive·`.bframe` 어디에도 복제되지 않는다.

### 5.2 스키마
```js
{
  "version": 1,
  "items": [
    {
      "id": "a7f3c2...",             // sha1(absolutePath) - 썸네일 파일명과 동일 키
      "path": "G:/공유 드라이브/.../EP03_SC012_v04.mp4",
      "name": "EP03_SC012_v04.mp4",
      "dir": "G:/공유 드라이브/.../EP03/애니메이션",
      "ext": ".mp4",
      "size": 128947200,             // bytes
      "duration": 24.5,              // seconds
      "thumbPath": "a7f3c2....jpg",  // recent-thumbs 폴더 기준 상대 경로, 없으면 null
      "openedAt": 1775676880000,     // ms epoch — 정렬 키
      "openCount": 7                 // 누적 열람 횟수, UI 미표시 (미래 확장용)
    }
  ],
  "pinned": ["a7f3c2...", "b9e4d1..."]  // 고정 항목 ID 배열 (최대 5)
}
```

### 5.3 정렬 규칙
```
list() 반환 순서:
  1. pinned 배열 순서 (사용자 고정 추가 순)
  2. 그 외 items 중 openedAt 내림차순
  3. 전체 표시 상한: 15개 (고정 5 + 일반 10)
```

### 5.4 쓰기 규칙
| 트리거 | 동작 |
|--------|------|
| `loadVideo()` 성공 | 같은 id 있으면 `openedAt` 갱신 + `openCount++`, 없으면 신규 삽입 |
| 신규 삽입 후 비고정 항목 > 10 | `openedAt` 가장 오래된 비고정 항목 제거 (고정은 절대 트림 X) |
| 썸네일 캡처 성공 | 해당 item의 `thumbPath` 갱신, `recent:updated` 이벤트 방출 |
| 파일 경로 검증 실패 | items에서 제거, 해당 id가 pinned에 있으면 pinned에서도 제거 |

### 5.5 동시성
- `electron-store`의 쓰기는 원자적(임시 파일 → rename). 추가 lock 불필요.
- 썸네일 쓰기도 `recent-thumbs/{id}.jpg.tmp` → rename 패턴 동일 적용.

### 5.6 ID 설계 근거
- **경로 기반 SHA1**: 같은 파일을 다시 열면 같은 id → 정렬/갱신 단순
- **파일 이동 시**: 새 경로는 새 id → 새 항목. 구 항목은 pruneMissing에서 정리
- **콘텐츠 해시 미사용**: 대용량 비디오에 비싸고 이득 적음

## 6. 아키텍처

### 6.1 책임 분리
각 모듈은 한 가지 역할만 담당하고, 다른 모듈의 내부를 몰라도 사용할 수 있다.

| 모듈 | 위치 | 역할 |
|------|------|------|
| `recent-files-store.js` | main | 메타데이터 CRUD, 원자적 저장, 용량 제한 |
| `recent-thumb-capture.js` | main | ffmpeg로 대표 프레임 1장 추출·저장 |
| IPC 핸들러 블록 | main/ipc-handlers.js | 위 두 모듈을 렌더러에 노출 |
| `recent` preload API | preload | IPC 래핑 |
| `recent-files-manager.js` | renderer | IPC 호출 래핑, 이벤트 발행, 캐시 |
| `recent-files-view.js` | renderer | DOM 렌더링 (비즈니스 로직 없음) |
| `recent-files.css` | renderer/styles | 시각 스타일 |

### 6.2 데이터 흐름
```
사용자가 파일 열기
   ↓
app.js: loadVideo() 성공
   ↓
recent-files-manager.add(filePath, metadata)  ← renderer
   ↓ (IPC)
main: recent-files-store.upsert(item) + 용량 트림
   ↓
main: recent-thumb-capture.captureFrame(videoPath) — 비동기
   ↓ (성공 시)
store 갱신 → recent:updated 이벤트 브로드캐스트
   ↓ (IPC)
renderer: recent-files-manager 'updated' 이벤트
   ↓
recent-files-view 재렌더링
```

### 6.3 IPC 채널 목록
| 채널 | 입력 | 출력 | 설명 |
|------|------|------|------|
| `recent:list` | - | `RecentItem[]` | 정렬된 전체 목록 |
| `recent:add` | `{path, name, dir, ext, size, duration}` | `{id, item}` | 업서트 |
| `recent:remove` | `id` | `{success}` | 개별 제거 |
| `recent:clear` | - | `{success}` | 전체 지우기 |
| `recent:togglePin` | `id` | `{success, pinned: boolean}` | 고정 토글 (5개 초과 시 reject) |
| `recent:pruneMissing` | - | `{removedIds: string[]}` | 존재하지 않는 경로 일괄 정리 |
| `recent:openInFolder` | `path` | `{success}` | `shell.showItemInFolder` |
| `recent:captureThumb` | `videoPath, id` | `{thumbPath?: string}` | 썸네일 캡처 및 저장 |
| `recent:updated` (이벤트) | - | `{items}` | 상태 변경 브로드캐스트 |

### 6.4 보안 고려
- IPC 핸들러는 기존 `validateFilePath` 패턴 재사용 (허용 확장자 검증)
- 렌더러는 직접 파일시스템 접근 없음. 썸네일도 main이 제공하는 로컬 URL로 로드.
- 저장소 로드 실패는 조용히 빈 배열로 폴백 (DoS 방지)

## 9. 테스트 방법

### 9.1 수동 테스트 시나리오
1. **기본 추가**: 새 영상 파일 열기 → 다시 앱 시작 → 목록에 해당 항목 보임 (썸네일 포함)
2. **중복 열기**: 같은 파일 여러 번 열기 → 하나의 항목만 존재, 시간만 갱신
3. **용량 트림**: 11개째 비고정 항목 열기 → 가장 오래된 비고정 항목이 사라짐
4. **고정**: 임의 항목 고정 → 재시작 후에도 상단 유지, 자동 트림 대상 제외
5. **고정 한계**: 6번째 고정 시도 → 거부 + 토스트
6. **파일 이동**: 목록에 있는 파일을 OS에서 삭제 → 회색 처리 + 클릭 시 제거 확인
7. **전체 지우기**: 빈 상태 우측 하단 버튼 → 섹션 전체 사라짐
8. **헤더 드롭다운**: 파일 열린 상태에서 `clock` 버튼 클릭 → 패널 표시, Esc 닫힘, 바깥 클릭 닫힘
9. **드래그앤드롭 호환**: 드롭존 위 리스트 영역에서 파일 드롭 → 정상 로드
10. **오디오 파일**: mp3/wav 열기 → 썸네일 대신 `music` 아이콘 표시

### 9.2 자동화 가능 부분
- `recent-files-store.js`의 CRUD·용량 트림·정렬 로직: 단위 테스트 (파일시스템 모킹)
- ID 생성 일관성: 동일 경로 → 동일 해시
- `recent-files-manager.js`의 이벤트 발행: IPC 모킹 테스트

### 9.3 회귀 확인
- 기존 파일 열기 흐름 (버튼 / 드래그앤드롭 / 파일명 클릭) 모두 정상 동작
- 버전 드롭다운, 재생목록 사이드바, 협업자 표시기 영향 없음

## 10. 열린 질문 (향후 결정)

현재 스펙에 포함되지 않았지만 구현 중 결정이 필요할 수 있는 사항:

1. **상대 시각 포맷의 정확한 문자열** — "10분 전 / 1시간 전 / 어제 / 2일 전 / 지난주 / 2주 전 / 1개월 전 / 오래전" 중 어디까지 구체화할지
2. **"더 보기" 클릭 시 전개/접기 애니메이션 여부** — 시각 일관성 관점에서 결정
3. **되돌리기 토스트의 정확한 대기 시간** — 현재 3초로 가정
4. **드롭다운 패널 너비 반응형 대응** — 창 폭이 매우 좁을 때의 대체 동작

이 항목들은 구현 시 기본값으로 진행하되, 팀 피드백 후 조정할 수 있도록 한 곳에 상수로 모아둔다.
