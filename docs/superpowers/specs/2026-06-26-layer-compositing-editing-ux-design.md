# Layer Compositing Editing UX Design

## Goal

BAEFRAME의 임시 레이어 합성 기능을 실제 편집처럼 다룰 수 있게 한다. 사용자는 이미지/영상 레이어를 복제하고, 순서를 조정하고, 크기 비율을 유지한 채 배치하고, 누락된 원본 파일을 다시 연결할 수 있어야 한다.

## Approved Scope

- 레이어 목록에 복제, 위로, 아래로, 비율 잠금 버튼을 추가한다.
- 기존 드래그 순서 변경, 선택, 삭제, 숨김 기능은 유지한다.
- 선택된 레이어는 목록, 프리뷰, 타임라인에서 더 뚜렷하게 보인다.
- 프리뷰에서 이동하거나 리사이즈할 때 화면 가장자리와 중앙선에 스냅한다.
- 리사이즈는 레이어별 비율 잠금 상태를 따른다. 새 레이어의 기본값은 비율 잠금 켜짐이다.
- 파일이 사라진 레이어는 누락 상태와 다시 연결 버튼을 보여준다.
- 저장 후 다시 열어도 레이어 순서, 비율 잠금, 위치, 크기, 시간 범위가 유지된다.

## Out of Scope

- 최종 합성 영상 export
- 회전, 정렬 프리셋, 키프레임
- 오디오 믹싱
- 공유용 미디어 폴더 패키징
- 위치/크기/시작/끝 시간 숫자 입력 인스펙터

## Data Model

`compositionLayers[]`에 선택 상태는 저장하지 않는다. 편집 상태 중 저장할 값은 기존 필드와 새 `aspectLocked: boolean`이다.

`missing`은 런타임 상태다. 파일 존재 확인 결과로 표시만 하고 저장 데이터에는 포함하지 않는다.

## UI Behavior

레이어 행은 다음 행동을 제공한다.

- 숨김/표시 토글
- 이름 변경
- 위/아래 이동
- 복제
- 비율 잠금 토글
- 파일 누락 시 다시 연결
- 삭제

복제된 레이어는 원본 바로 위 레이어 순서에 들어가며, 이름은 기존 중복 규칙을 따른다. 위치는 원본과 거의 겹치되 약간 이동해 새 레이어임을 알아볼 수 있게 한다.

스냅은 프리뷰의 왼쪽, 가운데, 오른쪽과 위, 가운데, 아래 기준으로 동작한다. 스냅 중에는 가이드 라인을 잠깐 표시한다.

## Error Handling

- 다시 연결 파일이 이미지/영상이 아니면 변경하지 않고 경고 토스트를 띄운다.
- 파일 선택을 취소하면 기존 레이어를 그대로 둔다.
- 누락 파일 확인 API가 실패하면 기존처럼 누락으로 단정하지 않는다.

## Testing

- 레이어 복제, 위/아래 이동, 비율 잠금 저장, 스냅 계산, 누락 파일 다시 연결을 Node 테스트로 검증한다.
- 런타임 소스 테스트로 버튼 액션, 스냅 가이드 스타일, 저장 연결이 빠지지 않았는지 확인한다.
- 기존 mpv overlay mirror 테스트를 유지해 별도 overlay window와 renderer 상태가 계속 맞는지 확인한다.
