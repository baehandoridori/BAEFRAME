# Moho 라벨 기반 컷 묶음 설계

작성일: 2026-05-23

## 목표

Moho에서 여러 씬을 한 영상으로 출력한 뒤, Premiere에서 컷별로 다시 자르는 과정을 BAEFRAME 안에서 줄인다.

BAEFRAME은 실제 영상 파일을 자르지 않는다. 대신 Moho 라벨 정보가 담긴 `info.txt`를 읽어서, 긴 출력 영상을 `a019`, `a020`, `sc023` 같은 가상 컷으로 나눠 보여준다. 사용자는 컷 단위로 이동하고 댓글을 남기며 리뷰할 수 있다.

완성본의 목표는 컷 전환 공백을 최소화하는 것이다. 다만 1차 프로토타입은 재생 속도 최적화보다 컷 배치 정확성을 먼저 검증한다.

## 사용자 흐름

상단 툴바에 `컷 묶음` 버튼을 추가한다. 이 버튼은 기존 `재생목록`과 분리된 별도 진입점이다.

사용자가 `컷 묶음`을 누르면 컷 묶음 전용 패널이 열린다. 첫 화면에는 큰 `영상 + info txt 쌍 추가` 버튼을 보여준다.

쌍 추가 흐름은 다음 순서다.

1. 사용자가 Moho 라벨 정보가 담긴 `info.txt`를 선택한다.
2. BAEFRAME이 txt를 읽고 컷 후보를 미리 보여준다.
3. 사용자가 해당 txt에 연결할 출력 영상을 선택한다.
4. BAEFRAME이 이 쌍을 컷 묶음에 추가한다.

여러 쌍을 추가할 수 있다. 예를 들어 캐릭터 A 출력본과 캐릭터 B 출력본을 각각 txt와 연결하면, BAEFRAME은 씬 번호 기준으로 컷을 섞어 하나의 컷 묶음으로 보여준다.

컷 묶음은 `.bcutlist` 파일로 저장한다. `.bcutlist`를 열면 같은 컷 묶음이 다시 열리고, 영상 파일이 모두 존재하면 바로 리뷰 화면으로 들어간다. 영상 경로가 깨진 컷은 회색으로 표시하고, 클릭하면 해당 출력 영상을 다시 연결하도록 안내한다.

## 파일 포맷

새 공유 파일 확장자는 `.bcutlist`다. 기존 `.bplaylist`와 섞지 않는다.

`.bcutlist`에는 다음 정보를 저장한다.

- 출력 영상 경로
- 원본 info txt 경로
- 원본 txt 내용 또는 읽어낸 결과
- 컷 목록
- 컷 정렬 순서
- 누락 씬 표시 옵션
- 향후 드로잉 기능을 붙일 수 있는 확장 필드

기본 저장 위치는 첫 번째 info txt가 있는 폴더다. 기본 파일명은 txt 또는 컷 범위 기반으로 제안한다. 예: `a019-a029.bcutlist`.

## txt 읽기 규칙

Moho txt 예시는 다음 형태다.

```txt
FPS: 24.0
Document Range: 1 - 204

03. sc023
    Range: 42 - 92
    Frames: 51f
```

BAEFRAME은 씬 번호 형태의 라벨만 컷으로 인정한다.

- `a020`: 인정
- `sc019`: 인정
- `실제 사운드 시작지점`: 제외

화면 표시 이름은 원본 라벨을 그대로 사용한다. 예를 들어 `sc019`는 목록에서도 `sc019`로 표시한다. 단, 정렬과 매칭은 숫자 기준으로 처리한다. `sc019`와 `a019`는 모두 19번 씬으로 본다.

같은 씬 번호가 여러 번 나오면 긴 구간을 우선한다. 예를 들어 `a028 93-204`와 `a028 205-205`가 함께 있으면 1프레임짜리 중복은 자동 제외한다.

씬 형태가 아닌 라벨이나 자동 제외된 중복은 경고창으로 막지 않는다. 대신 `무시된 라벨 2개`처럼 작게 요약하고, 펼치면 어떤 항목이 왜 제외됐는지 확인할 수 있게 한다.

## 프레임 규칙

실제 확인 결과, Moho txt는 1부터 시작하고 BAEFRAME 영상 프레임은 0부터 시작한다.

```txt
Moho/txt Range: 42 - 92
BAEFRAME Range: 41 - 91
```

변환 규칙은 다음과 같다.

- BAEFRAME 시작 프레임 = txt 시작 프레임 - 1
- BAEFRAME 끝 프레임 = txt 끝 프레임 - 1
- 끝 프레임도 해당 컷에 포함한다

컷 목록의 기본 프레임 표시는 BAEFRAME 기준으로 한다. 상세 정보에서는 Moho/txt 원본 범위도 같이 보여준다.

예:

- 기본: `41 - 91f`
- 상세: `Moho 42 - 92f`

## 컷 목록과 순서

컷 묶음 패널의 기본 목록은 간단하게 보인다.

- `sc019`
- `a020`
- `a022`

각 컷을 펼치면 다음 정보를 보여준다.

- 원본 출력본
- BAEFRAME 프레임 범위
- Moho/txt 원본 프레임 범위
- 컷 길이
- 제외된 라벨 요약

전체 순서는 기본적으로 씬 번호 숫자순이다. 예외가 있으면 사용자가 수동으로 순서를 바꿀 수 있고, 이 수동 순서는 `.bcutlist`에 저장한다.

숫자 사이에 빠진 씬은 기본적으로 숨긴다. 옵션을 켜면 `a021 없음`처럼 누락 씬을 표시한다.

프로토타입에는 검색과 필터를 넣지 않는다.

## 리뷰 화면 동작

컷을 클릭하면 해당 컷의 시작점으로 이동만 한다. 자동 재생은 하지 않는다. 사용자는 Space 또는 재생 버튼으로 직접 재생한다.

재생 중 한 컷의 끝에 도달하면 멈추지 않고 다음 컷으로 이어서 재생한다. 컷은 탐색과 댓글 기준이고, 재생은 자연스럽게 이어지는 것이 목표다.

현재 컷은 두 곳에 표시한다.

- 영상 좌상단 오버레이에 현재 컷 이름 표시
- 타임라인에서 현재 컷 구간을 다른 색으로 강조

타임라인에는 컷 구간 블록과 컷 이름 라벨을 표시한다.

## 댓글과 저장

`.bcutlist`는 컷 나누기 정보를 저장한다. 댓글은 원본 출력 영상의 기존 `.bframe`에 저장한다.

예를 들어 컷 묶음에서 `a020`에 댓글을 달아도, 실제 저장은 `a020`이 포함된 출력 영상의 `.bframe`에 들어간다. 그래서 같은 출력 영상을 일반 BAEFRAME 리뷰로 열어도 댓글이 보이고, 컷 묶음으로 열어도 해당 컷 위치에 댓글이 보인다.

출력 영상에 `.bframe`이 없으면, 컷 묶음에서 댓글을 처음 달 때 자동으로 생성한다.

댓글 목록은 컷 묶음 모드에서만 컷 이름과 전체 시간을 함께 표시한다.

예:

```txt
a020 00:01:12 · 전체 00:08:20
```

일반 영상 리뷰에서는 기존처럼 시간 기준으로만 표시한다.

드로잉은 프로토타입 화면 기능에 포함하지 않는다. 다만 나중에 붙일 수 있게 저장 구조를 막지 않는다.

## 오류 처리

영상 파일 경로가 깨졌을 때는 컷 목록을 삭제하지 않는다. 해당 영상에서 온 컷들을 회색 또는 비활성 상태로 표시한다. 사용자가 해당 컷을 클릭하면 영상 다시 연결을 요청한다.

txt에서 읽을 수 없는 항목은 컷 후보에서 제외하고 요약에 표시한다.

`.bcutlist`를 열 때 영상 파일이 모두 존재하면 바로 리뷰 화면으로 들어간다. 문제가 있으면 검수 또는 다시 연결 화면을 보여준다.

## 프로토타입 완료 기준

프로토타입은 다음까지 완료해야 한다.

- 상단 툴바에 `컷 묶음` 버튼 추가
- 별도 컷 묶음 패널 추가
- `info txt 선택 → 출력 영상 선택`으로 쌍 추가
- txt에서 씬 번호 라벨만 읽기
- `txt 프레임 - 1` 규칙으로 BAEFRAME 프레임 계산
- 중복 씬은 긴 구간 우선
- 누락 씬 표시 옵션
- 컷 목록 숫자순 정렬
- 사용자가 컷 순서를 수동 변경하고 저장
- 컷 클릭 시 맞는 영상과 프레임으로 이동
- 영상 좌상단 현재 컷 오버레이
- 타임라인 컷 블록, 컷 이름 라벨, 현재 컷 강조
- 컷 묶음 모드 댓글 표시
- 댓글은 원본 영상 `.bframe`에 저장
- `.bcutlist` 저장과 열기
- 영상 경로가 깨졌을 때 회색 표시와 다시 연결

## 2차 개선 범위

2차 개선은 전환 지연 최소화다.

- 다음 영상 미리 준비
- 파일 전환 시 공백 줄이기
- 기존 타임라인 이어붙이기보다 더 자연스럽게 만들기
- 필요하면 캐시와 프리로드 강화

## 테스트 기준

샘플 txt 2개와 출력 영상 2개를 기준으로 검증한다.

검증 항목:

- txt에서 `a020`, `sc023` 같은 씬 라벨만 컷으로 읽는지
- `실제 사운드 시작지점` 같은 메모 라벨을 제외하는지
- 1프레임 중복을 긴 구간 우선으로 제외하는지
- `txt 42-92`가 BAEFRAME `41-91`로 계산되는지
- 컷 목록이 숫자순으로 섞이는지
- 누락 표시 옵션을 켰을 때 빈 씬이 보이는지
- 컷 클릭 시 맞는 출력 영상과 프레임으로 이동하는지
- 댓글 작성 시 원본 영상 `.bframe`에 저장되는지
- `.bcutlist` 저장 후 다시 열어도 같은 컷 묶음이 복원되는지
- 영상 경로를 일부러 깨뜨렸을 때 회색 표시와 다시 연결 흐름이 나오는지
