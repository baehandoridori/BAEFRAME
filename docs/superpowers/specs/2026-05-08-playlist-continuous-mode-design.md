# BAEFRAME 재생목록 이어보기 및 토스트 안정화 설계

작성일: 2026-05-08

## 1. 목적

재생목록의 영상이 사용자가 기대한 순서대로 안정적으로 재생되고, 여러 영상을 하나의 흐름처럼 이어서 볼 수 있게 한다.

사용자가 겪은 주요 문제는 다음과 같다.

- 재생목록에 넣은 영상이 가끔 재생되지 않음
- 재생목록 순서가 뒤죽박죽처럼 느껴짐
- 여러 영상을 한 번에 쭉 이어서 보고 싶음
- 토스트 알림이 여러 개 쌓였다 사라질 때 앞으로 튀거나 화면을 벗어나는 문제가 있음

이번 작업은 기존 BAEFRAME의 어두운 작업 도구 톤을 유지하면서, 재생목록 UX를 안정화하는 데 집중한다.

## 2. 확정된 방향

접근 방식은 "안정화 중심형"이다.

최소 수정으로 덧붙이기만 하지 않고, 재생목록 순서, 이어보기 상태, 오류 건너뛰기, 백그라운드 준비, 토스트 스택을 필요한 만큼 정리한다. 반대로 컷 편집이나 통합 타임라인 직접 편집처럼 저장 구조를 크게 바꾸는 기능은 이번 범위에서 제외한다.

## 3. 화면 흐름

재생목록 패널 상단에 `리뷰` / `이어보기` 탭을 둔다.

- `리뷰` 탭: 기존처럼 영상 하나를 열어 댓글과 드로잉을 확인하고 수정한다.
- `이어보기` 탭: 같은 재생목록을 사용하되, 하단 타임라인을 여러 영상을 하나로 이어붙인 흐름처럼 보여준다.

이어보기 재생은 현재 선택된 영상부터 시작한다. 선택된 영상이 없으면 첫 번째 영상부터 시작한다.

현재 플레이어가 선택된 영상을 이미 열고 있으면 현재 시간부터 시작한다. 다른 영상이 열려 있거나 아직 열리지 않았다면 선택된 영상의 처음부터 시작한다.

마지막 영상까지 끝나면 기본은 멈추고 `재생목록 재생 완료` 토스트를 보여준다. 재생목록 설정에서 반복 재생을 켜면 마지막 영상 뒤에 첫 번째 영상으로 돌아간다.

이어보기 중 `리뷰` 탭으로 전환하면 이어보기는 멈추고, 현재 영상과 현재 시간에서 리뷰 모드로 전환한다. 이렇게 해야 댓글을 보거나 수정하려는 순간 자동으로 다음 영상으로 넘어가지 않는다.

## 4. 정렬과 순서 안정화

새 영상을 추가하면 기본으로 파일명 자연 정렬을 적용한다.

예: `1, 2, 10` 순서가 `1, 10, 2`처럼 섞이지 않고 `1, 2, 10`으로 들어가야 한다.

사용자가 고를 수 있는 정렬 기준은 다음 세 가지다.

- 파일명순
- 추가한 순서
- 수정한 날짜순

사용자가 드래그로 순서를 바꾸면 그 수동 순서가 우선한다. 이후 새 영상을 추가하면 기존 수동 순서는 건드리지 않고, 새 영상들만 파일명순으로 정렬해서 끝에 붙인다.

이어보기 중 순서를 드래그하거나 정렬 버튼을 누르면, 현재 보고 있는 영상은 끊지 않는다. 다음 영상부터 새 순서를 따른다.

재생목록 파일에는 재생목록별 설정을 저장한다. 반복 재생, 정렬 기준, 수동 순서가 재생목록마다 따로 기억되어야 한다.

## 5. 이어보기 준비와 오류 처리

이어보기를 시작하면 먼저 빠른 검사를 한다.

- 파일이 실제로 존재하는지 확인
- 지원 가능한 파일 형식인지 확인
- 문제 있는 항목은 재생목록 안에 표시

문제가 있어도 전체 재생을 막지 않는다. 재생 가능한 영상부터 바로 시작한다.

재생 중에는 다음 영상을 미리 준비한다. 변환이 필요한 영상은 백그라운드에서 준비한다.

재생목록 항목에는 작은 상태 표시를 둔다.

- 준비 중
- 준비 완료
- 건너뜀
- 문제 있음

재생목록 상단에는 전체 준비 상태를 보여준다.

예: `3/12개 준비됨`

변환 중인 영상 차례가 왔는데 아직 준비가 끝나지 않았으면 최대 5초만 기다린다. 5초 안에 준비되면 재생하고, 끝나지 않으면 토스트로 알린 뒤 다음 영상으로 넘어간다.

문제 영상이 여러 개 연속으로 나오면 토스트를 여러 개 띄우지 않고 하나로 묶는다.

예: `3개 영상을 건너뜀`

각 문제 항목은 재생목록 안에 표시가 남아야 한다.

## 6. 통합 타임라인

`이어보기` 탭의 타임라인은 재생목록의 모든 영상을 전체 길이 그대로 이어붙인 하나의 흐름처럼 보여준다.

각 영상의 경계는 평소에는 얇은 선으로만 보인다. 마우스를 올리면 파일명과 시간 정보가 나타난다.

통합 타임라인의 특정 구간을 클릭하면 해당 영상과 그 시간으로 바로 이동한다.

이번 1차 구현에서는 영상별 시작/끝 구간을 자르는 기능을 넣지 않는다. 각 영상은 전체 길이 그대로 이어보기 대상이 된다.

## 7. 통합 타임라인 댓글 표시

통합 타임라인에는 재생목록 안의 모든 영상 댓글 구간을 표시한다.

댓글 필터와 레이어 표시 상태는 기존 댓글 패널의 설정을 그대로 따른다. 사용자가 댓글 패널에서 숨긴 항목은 통합 타임라인에서도 숨겨져야 한다.

댓글이 적으면 기존 댓글 타임라인처럼 여러 줄로 나눠서 겹치지 않게 표시한다. 너무 빽빽할 때만 숫자 배지로 묶고, 마우스를 올리면 내용을 확인할 수 있게 한다.

댓글 막대를 클릭하면 해당 영상과 시간으로 이동한다. 댓글 수정은 기존 댓글 패널에서만 한다.

구현상 주의할 점:

- 각 영상의 댓글 데이터는 영상별 `.bframe` 파일에 따로 저장되어 있다.
- 통합 타임라인은 다른 영상의 `.bframe`을 읽기 전용으로 모아 보여주는 별도 인덱스를 가져야 한다.
- 다른 영상의 댓글을 표시하기 위해 현재 영상의 `commentManager` 상태를 덮어쓰면 안 된다.
- 댓글 위치 계산은 영상별 로컬 시간을 전체 재생목록 시간으로 변환해서 표시한다.
- 서로 다른 FPS를 가진 영상이 섞일 수 있으므로 내부 계산은 초 단위 시간을 기준으로 두는 것이 안전하다.

## 8. 토스트 안정화

기존 토스트의 어두운 톤과 겹쳐 쌓이는 느낌은 유지한다.

고쳐야 할 문제는 다음과 같다.

- 여러 토스트가 사라질 때 앞으로 튀어나와 보이는 문제
- 펼쳐진 토스트가 창 밖으로 넘어가거나 잘리는 문제
- 토스트 위에 마우스가 있어도 다른 토스트의 자동 사라짐 시간이 계속 흐르는 문제

수정 방향:

- 새 토스트와 오래된 토스트의 순서를 한 기준으로 통일한다.
- 사라지는 중인 토스트가 다른 토스트보다 위로 올라오지 않게 한다.
- 토스트 컨테이너에 마우스를 올리면 모든 토스트의 자동 사라짐을 잠시 멈춘다.
- 숨겨진 토스트를 펼쳐 볼 때 화면을 넘지 않고, 토스트 영역 안에서 스크롤되게 한다.
- 이어보기 오류 토스트는 연속 실패를 묶어서 보여준다.

## 9. 주요 변경 예상 파일

구현 전에 다시 확인해야 하지만, 현재 코드 조사 기준으로 주요 변경 후보는 다음과 같다.

- `renderer/scripts/modules/playlist-manager.js`
  - 정렬, 수동 순서, 준비 상태, 이어보기 상태 관리
- `renderer/scripts/app.js`
  - 재생목록 UI 이벤트, 이어보기 시작/정지, 영상 전환, 오류 토스트 연결
- `renderer/scripts/modules/timeline.js`
  - 통합 타임라인 세그먼트와 통합 댓글 표시
- `renderer/styles/playlist-panel.css`
  - `리뷰` / `이어보기` 탭, 준비 상태, 통합 타임라인 스타일
- `renderer/styles/main.css`
  - 토스트 스택, 펼침, 스크롤, hover pause 시각 상태
- `shared/playlist-schema.js`
  - 재생목록별 설정과 하위 호환 저장 구조

## 10. 데이터 저장 설계

기존 `.bplaylist` 파일은 계속 열 수 있어야 한다.

새 설정은 재생목록별로 저장한다.

예상 저장 항목:

- 이어보기 반복 재생 여부
- 마지막 선택 정렬 기준
- 수동 순서 여부
- 각 항목의 순서 값

준비 중, 준비 완료, 건너뜀 같은 상태는 실행 중 상태로 다루는 것이 기본이다. 이 상태는 앱을 다시 열면 다시 검사해서 갱신한다. 단, 파일 없음처럼 명확한 문제는 다음 검사에서 다시 표시할 수 있어야 한다.

## 11. 테스트 기준

실사용자 기준 테스트:

- 영상을 여러 개 추가했을 때 파일명순으로 자연스럽게 들어가는지 확인
- 드래그로 순서를 바꾼 뒤 새 영상을 추가해도 기존 순서가 유지되는지 확인
- 이어보기 탭에서 현재 선택된 영상부터 재생되는지 확인
- 마지막 영상이 끝나면 기본적으로 멈추고, 반복 설정을 켜면 첫 영상으로 돌아가는지 확인
- 문제 영상이 있어도 재생 가능한 영상은 계속 이어지는지 확인
- 문제 영상이 여러 개일 때 토스트가 묶여서 보이는지 확인
- 토스트가 여러 개 쌓이고 사라져도 앞으로 튀거나 화면 밖으로 나가지 않는지 확인
- 토스트 영역에 마우스를 올렸을 때 자동 사라짐이 멈추는지 확인
- 통합 타임라인에서 영상 경계와 댓글 구간이 보이는지 확인
- 통합 타임라인 댓글을 클릭하면 해당 영상과 시간으로 이동하는지 확인

개발자 기준 테스트:

- 기존 `.bplaylist` 파일을 열어도 오류가 없어야 한다.
- 기존 댓글, 드로잉, 리뷰 저장 흐름이 깨지면 안 된다.
- 지원하지 않는 파일, 삭제된 파일, 변환 실패 파일을 포함한 재생목록에서 이어보기가 멈추지 않아야 한다.
- 자동 재생 종료 이벤트가 리뷰 모드와 이어보기 모드를 혼동하지 않아야 한다.
- 통합 댓글 인덱스가 다른 영상의 댓글 데이터를 읽어도 현재 영상의 comment manager를 덮어쓰지 않아야 한다.
- 토스트 제거 애니메이션 중 DOM 순서와 z-index가 안정적으로 유지되어야 한다.

## 12. 다음 세션 인계 항목

이번 범위에서 제외하지만, 다음 세션에서 이어서 검토할 항목은 다음과 같다.

1. 영상별 시작/끝 구간 자르기
   - 재생목록이 단순 이어보기를 넘어 편집 타임라인에 가까워진다.
   - 저장 구조에 clip start/end 값이 필요하다.

2. 통합 타임라인에서 댓글 직접 편집
   - 이번에는 댓글 막대를 클릭하면 해당 영상으로 이동하고 기존 댓글 패널에서 수정한다.
   - 직접 편집을 넣으려면 어떤 영상의 `.bframe`을 수정하는지 명확히 고정해야 한다.

3. 이어보기에서 임시 제외
   - 이번에는 정상 영상은 모두 이어보기 대상이다.
   - 다음 단계에서는 항목별 `이어보기에서 제외` 토글을 고려할 수 있다.

4. 영상 길이순, 폴더별 정렬
   - 이번 정렬 기준은 파일명순, 추가한 순서, 수정한 날짜순이다.
   - 더 복잡한 정렬은 실제 사용 필요가 확인된 뒤 추가한다.

5. 마지막 이어보기 위치 기억
   - 이번에는 선택된 영상부터 시작한다.
   - 다음 단계에서는 재생목록별 마지막 이어보기 위치 저장을 검토할 수 있다.

6. 정밀 검사 자동화 범위 확대
   - 이번에는 기본 빠른 검사와 사용자가 요청하는 정밀 검사를 구분한다.
   - 다음 단계에서는 큰 재생목록에서 정밀 검사를 언제 자동으로 제안할지 검토한다.

## 13. 승인 상태

이 설계는 사용자와 문답을 통해 다음 선택을 확정한 결과다.

- 이어보기는 별도 탭으로 제공
- 통합 타임라인은 모든 영상을 하나의 흐름처럼 표시
- 댓글은 전체 영상의 댓글을 표시하되, 수정은 기존 댓글 패널에서 수행
- 마지막 영상 후 기본은 정지, 설정으로 반복 재생 가능
- 오류 영상은 건너뛰고 토스트는 연속 실패를 묶어서 표시
- 시작 전 검사는 빠르게, 정밀 검사는 별도 실행
- 변환이 필요한 영상은 백그라운드에서 준비
- 변환 대기는 최대 5초
- 토스트는 기존 스타일을 유지하되 스택, hover pause, 펼침 스크롤을 안정화
