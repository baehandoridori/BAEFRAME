# Fabric 드로잉 브러시 설정 1단계 설계

- 작성일: 2026-07-21
- 상태: 사용자 승인 완료, 구현 전 확정본
- 복귀 기준: `94048cdf7da0f5e7d1ad5b0a5e563103f698bb57`
- 작업 브랜치: `codex/baeframe-fabric-brush-style-v1`
- 상위 설계: `docs/superpowers/specs/2026-07-17-fabric-drawing-surface-v3-design.md`

## 1. 결정 요약

사용자가 실제 앱에서 괜찮다고 확인한 Fabric 시험판 지점으로 돌아가 첫 번째 기능만 추가한다.

이번 단계는 새 Fabric 브러시에 다음 세 설정을 제공한다.

1. 기존 BAEFRAME과 같은 8색 팔레트
2. 1–50px 브러시 크기
3. 10–100% 불투명도

설정은 MPVOverlayHost 안의 Fabric 런타임 메모리에만 존재한다. B를 껐다 켜는 동안에는 유지되지만 리뷰 파일, 사용자 설정, 타임라인, 협업 데이터에는 기록하지 않는다.

## 2. 사용자 경험 계약

### 2.1 표시 위치

- 기존 Fabric 시험 툴바의 Brush, V, Delete, Clear 옆에 `브러시 설정` 버튼 하나를 추가한다.
- 버튼에는 현재 색상 점과 `3px · 100%` 형식의 요약을 표시한다.
- 버튼을 누르면 툴바 아래에 붙은 최대 360px 너비의 설정 패널이 열리고, 다시 누르면 닫힌다.
- 팔레트와 두 슬라이더는 이 패널 안에만 배치해 기본 툴바가 길어지지 않게 한다.
- B로 드로잉 모드를 켰을 때만 기존 툴바와 함께 보인다.
- B로 끄면 기존 동작대로 툴바 전체가 숨고 영상 입력이 복원된다.
- 설정 패널의 열림 상태와 값은 같은 앱 실행 중 B를 껐다 켜도 유지한다.
- 별도 BrowserWindow, 토스트, 모달 창은 만들지 않는다.

### 2.2 색상

기존 BAEFRAME의 익숙한 팔레트를 같은 순서와 값으로 제공한다.

| 이름 | 값 |
|---|---|
| 빨강 | `#ff4757` |
| 노랑 | `#ffd000` |
| 초록 | `#26de81` |
| 파랑 | `#4a9eff` |
| 하양 | `#ffffff` |
| 검정 | `#000000` |
| 민트 | `#1abc9c` |
| 핑크 | `#ff6b9d` |

- 기본값은 빨강 `#ff4757`이다.
- 색상 버튼은 최소 40px 입력 영역과 22px 색상 점을 사용한다.
- 현재 색상은 시각적 강조와 `aria-pressed="true"`로 함께 표현한다.
- 하양 색상 점에는 어두운 영상 위에서도 경계를 볼 수 있는 테두리를 둔다.
- 이번 단계에는 사용자 지정 색상, 스포이트, 최근 색상은 넣지 않는다.

### 2.3 크기

- 범위: 1–50px
- 간격: 1px
- 기본값: 3px
- 슬라이더 옆에 현재 값을 `3px` 형식으로 즉시 표시한다.
- 현재 크기를 알아볼 수 있는 원형 미리보기를 함께 표시한다.
- 슬라이더 양옆에 1px씩 줄이고 늘리는 `−`와 `+` 버튼을 제공한다.
- 포인터 사용자는 슬라이더를 직접 끌고, 키보드 사용자는 Tab으로 `−`/`+`에 이동한 뒤 Enter로 조절할 수 있다.

### 2.4 불투명도

- 범위: 10–100%
- 간격: 1%
- 기본값: 100%
- 슬라이더 옆에 현재 값을 `100%` 형식으로 즉시 표시한다.
- Fabric 객체에는 0.1–1 범위의 숫자로 기록한다.
- 슬라이더 양옆에 1%씩 줄이고 늘리는 `−`와 `+` 버튼을 제공한다.
- 포인터 사용자는 슬라이더를 직접 끌고, 키보드 사용자는 Tab으로 `−`/`+`에 이동한 뒤 Enter로 조절할 수 있다.

MPVOverlayHost는 overlay에 포커스가 있을 때 방향키를 메인 창으로 전달한다. 이번 단계는 그 키 라우팅을 바꾸지 않는다. 따라서 range 방향키 조작에 의존하지 않고, 기존 재생 단축키 계약을 보존하면서도 Enter로 작동하는 증감 버튼을 키보드 경로로 사용한다.

### 2.5 적용 규칙

- 설정 변경은 새로 시작하는 획부터 적용한다.
- 이미 그린 획의 색상, 크기, 불투명도는 바뀌지 않는다.
- 포인터를 누른 순간 세 설정을 한 번 복사해 그 획에 고정한다.
- 획을 그리는 도중 설정 UI가 바뀌어도 진행 중인 획의 미리보기와 완성 결과는 서로 달라지지 않는다.
- B를 껐다 다시 켜거나 다른 프레임을 방문해도 같은 앱 실행 중에는 마지막 설정을 유지한다.
- 앱을 종료하거나 Fabric overlay host가 재생성되면 기본값으로 돌아가는 것은 이번 단계에서 정상이다.

## 3. 데이터와 렌더링

Fabric 런타임은 다음 형태의 로컬 브러시 상태 하나를 가진다.

```js
{
  color: '#ff4757',
  size: 3,
  opacity: 1
}
```

새 획 레코드의 `style`에는 세 값을 모두 남긴다.

```js
{
  color: '#ffd000',
  size: 24,
  opacity: 0.4
}
```

- `size`는 perfect-freehand 경로 생성에 사용되므로 경로 자체에 반영된다.
- `color`는 Fabric Path의 `fill`에 사용한다.
- `opacity`는 Fabric Path의 `opacity`에 사용한다.
- 미리보기와 확정 획 모두 같은 스타일 스냅샷을 사용한다.
- B 재진입 또는 프레임 재방문으로 장면을 다시 그릴 때 획 레코드의 스타일을 그대로 복원한다.
- 이 단계 이전에 만들어진 opacity 없는 로컬 레코드는 100%로 표시한다.

## 4. 상태 수명과 저장 경계

이번 설정은 다음에만 영향을 준다.

- MPVOverlayHost 내부 Fabric 시험 툴바
- 현재 앱 실행 중 새로 그리는 Fabric 획
- 현재 앱 실행 중 메모리 장면 복원

다음에는 절대 기록하거나 전달하지 않는다.

- `.bframe`의 `drawings`
- `.bframe`의 `drawingsV3`
- `ReviewDataManager`
- `user-settings.js`
- 협업 Broadcast
- 클립보드
- 타임라인 키프레임
- main renderer와의 새 IPC

색상·크기·불투명도를 바꾸는 것만으로는 장면의 `dirty`, `mutationCount`, `objectCount`, `saveAttemptCount`가 변하지 않아야 한다.

## 6. 변하지 않아야 하는 핵심 계약

1. B는 Fabric 입력과 툴바만 켜고 끈다.
2. B는 mpv/HTML5 엔진, 영상 파일, 재생 위치, 재생 상태, 타임라인을 바꾸지 않는다.
3. Fabric surface는 B 토글 때 생성하거나 파괴하지 않는다.
4. 영상 재생의 주인은 계속 하나다.
5. 브러시 설정과 선택만으로 저장을 시도하지 않는다.
6. V 선택·이동·Delete·Clear의 기존 시험판 동작은 유지한다.
7. 드로잉 종료 뒤 B와 Space가 계속 작동해야 한다.
8. 기존 배포 앱과 현재 실행 중인 다른 BAEFRAME 인스턴스는 건드리지 않는다.

## 7. 입력 정규화

- 크기는 정수가 아닌 값을 숫자로 변환한 뒤 1–50으로 제한한다.
- 크기 변환에 실패하면 직전 정상값을 유지한다.
- 불투명도 UI 값은 숫자로 변환한 뒤 10–100으로 제한한다.
- 불투명도 변환에 실패하면 직전 정상값을 유지한다.
- 획 레코드의 opacity가 없거나 유효하지 않으면 렌더링 값은 1이다.
- 팔레트 값은 코드에 정의된 8개 값만 버튼으로 제공하므로 자유 입력 색상 검증 경로는 만들지 않는다.

## 8. 자동 검증

구현은 반드시 테스트 우선으로 진행한다.

1. 새 동작을 표현하는 테스트를 먼저 작성한다.
2. 기존 코드에서 그 테스트가 예상 이유로 실패하는 것을 확인한다.
3. 그 테스트만 통과시키는 최소 구현을 작성한다.
4. 집중 테스트와 기존 112개 기준 테스트를 다시 실행한다.

새 테스트는 최소한 다음을 증명해야 한다.

- 정확한 기본값, 팔레트, 범위, 접근성 속성
- 크기·불투명도 증감 버튼이 상한과 하한을 넘지 않고 작동함
- 색상·크기·불투명도 변경이 새 획에 적용됨
- transient preview와 확정 획이 같은 스타일을 사용함
- 기존 획은 설정 변경으로 바뀌지 않음
- 진행 중인 획은 포인터 시작 시 스타일을 고정함
- B warm disable/enable 뒤 컨트롤 값과 획 스타일이 유지됨
- 잘못된 크기와 불투명도 입력이 안전하게 제한되거나 직전값으로 복귀함
- opacity 없는 이전 레코드가 100%로 렌더링됨
- 설정만 바꿀 때 장면 변경 및 저장 카운터가 증가하지 않음

## 9. 실제 앱 검증

자동 검증과 독립 코드 검토를 통과한 뒤 새 worktree에서만 패키징한다.

시험 앱은 다음 조건으로 실행한다.

- `--fabric-drawing-pilot`
- `--skip-shell-registration`
- 매번 고유한 `--multi-instance-profile=fabric-brush-style-<timestamp>`
- `BAEFRAME_MPV_PILOT=1`

사용자 확인 순서:

1. 영상을 재생한다.
2. B로 드로잉 모드에 진입한다.
3. 기본 빨강 3px 100% 획을 그린다.
4. 노랑 24px 40%로 바꾸고 두 번째 획을 그린다.
5. 두 획이 각각 원래 스타일을 유지하는지 본다.
6. B를 끄고 Space 재생 및 B 재진입을 확인한다.
7. V로 두 획을 선택·이동하고 Delete를 확인한다.
8. 재생 중 B를 20회 반복한다.
9. 깜빡임, 이중 재생, 타임라인 변화, 저장 실패 알림이 없는지 확인한다.

## 10. 실패와 되돌리기

- 구현 실패 시 `94048cd` 기준점은 수정하지 않는다.
- 첫 기능 브랜치만 고치거나 폐기한다.
- 기존 대형 `fabric-phase0b` 미완성 작업은 가져오거나 정리하지 않는다.
- 사용자 실제 검증을 통과하기 전에는 Undo/Redo나 다음 기능을 같은 브랜치에 추가하지 않는다.
