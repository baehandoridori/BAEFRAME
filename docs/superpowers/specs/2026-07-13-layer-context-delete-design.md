# 레이어 우클릭 삭제 설계

## 목적

드로잉 레이어 행을 우클릭했을 때 열리는 기존 설정 말풍선에서, 그 레이어를 바로 삭제할 수 있게 한다.

## 범위

- 기존 레이어 설정 말풍선 하단에 `레이어 삭제` 버튼을 추가한다.
- 버튼은 우클릭한 레이어 ID를 사용한다. 현재 활성 레이어와 다르더라도 활성 레이어를 삭제하지 않는다.
- 마지막 남은 레이어는 기존 삭제 규칙을 유지해 삭제하지 않고 안내를 표시한다.
- 삭제에 성공하면 말풍선을 닫고, 레이어 타임라인을 다시 렌더링하며 기존 삭제 토스트를 표시한다.

## 제외 범위

- 별도 우클릭 메뉴나 확인 대화상자는 추가하지 않는다.
- 레이어 삭제 규칙, 드로잉 데이터 형식, 단축키는 변경하지 않는다.
- 웹 뷰어와 컴포지팅 레이어 UI는 변경하지 않는다.

## 사용자 흐름

1. 사용자가 타임라인의 드로잉 레이어 행을 우클릭한다.
2. 기존 이름·색상·불투명도 설정 말풍선이 열린다.
3. 사용자가 하단의 `레이어 삭제`를 누른다.
4. 앱은 우클릭했던 레이어를 삭제하고, 말풍선을 닫아 갱신된 레이어 목록을 보여 준다.

## 오류 처리

- 우클릭 대상 레이어가 사라졌으면 삭제하지 않는다.
- 마지막 레이어 삭제 요청은 기존 경고 토스트를 표시하고 말풍선은 유지한다.

## 검증

- 소스 회귀 테스트로 우클릭 팝업의 삭제 버튼, 우클릭 대상 ID 전달, 마지막 레이어 보호를 검증한다.
- `npm run test:drawing`, `npm run lint`, `npm run build`를 실행한다.
- 빌드된 앱에서 우클릭한 비활성 레이어가 정확히 삭제되는지와 마지막 레이어 보호를 사람이 확인한다.
