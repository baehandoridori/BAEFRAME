# MPV Pilot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an opt-in mpv direct playback pilot while preserving the existing HTML5 and FFmpeg fallback behavior.

**Architecture:** A new main-process `MPVManager` owns mpv detection, launch, JSON IPC command dispatch, and shutdown. The renderer only talks through preload IPC and `VideoPlayer` delegates controls when external-engine mode is active. The pilot can be activated through a local app setting or `BAEFRAME_MPV_PILOT=1`.

**Tech Stack:** Electron main/renderer, Node `child_process`, Node `net`, mpv JSON IPC, Node test runner.

## Global Constraints

- MPV pilot only activates when the local app setting is enabled or `BAEFRAME_MPV_PILOT=1`.
- Existing HTML5 playback and FFmpeg transcode paths remain the default.
- Audio mode and split view stay on the existing path.
- Missing mpv is not an app error; it simply disables the pilot.
- MPV availability detection must work without activation so settings can show whether mpv is present.
