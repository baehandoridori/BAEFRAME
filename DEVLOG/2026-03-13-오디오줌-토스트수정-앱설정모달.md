# 오디오 파형 줌 + 토스트 수정 + 앱 설정 모달 개발

## 요약

| 항목 | 수정 파일 | 이유 | 우선순위 | 상태 |
|------|----------|------|---------|------|
| Phase 1: 파형 위치 | audio-waveform.js | 파형/플레이헤드를 화면 2/3 지점으로 | 높음 | ✅ 완료 |
| Phase 2: 파형 줌 | audio-waveform.js, main.css | 휠 줌 + 부드러운 애니메이션 + 줌% 표시 | 높음 | ✅ 완료 |

## 배경 및 목적

1. 오디오 파형이 화면 전체를 차지하여 적절한 위치 조정 필요
2. 오디오 파형 확대/축소 기능이 없어 세밀한 탐색 불가
3. 토스트 알림이 역순으로 쌓여 가장 최신 알림이 이전 알림 아래에 표시
4. 앱 전체 설정을 관리할 수 있는 통합 모달 필요
5. 알림 위치/지속시간 등 사용자 커스터마이징 지원

## Phase 1: 파형 위치 (h * 0.55 → h * 0.67)

`_drawWaveform()`과 `_drawOverlay()`의 `centerY`를 `h * 0.67`로 통일.
화면 2/3 지점에 파형 중심선과 플레이헤드 다이아몬드가 위치.

## Phase 2: 오디오 파형 휠 줌

### 구현
- `_zoom`, `_targetZoom`, `_scrollOffset` 상태 추가
- `_onWheel()`: 마우스 커서 기준 줌 인/아웃 (100%~800%)
- 부드러운 lerp 보간 (`_drawWaveform` 내에서 `_zoom += (_targetZoom - _zoom) * 0.15`)
- `_resampleFromRaw()`: `canvas.width * zoom`으로 전체 바 수 계산
- 보이는 범위만 렌더링 (`startBar` ~ `endBar`)
- `_seekFromEvent()`, `_onMouseMove()` 줌/스크롤 반영
- 줌 인디케이터 (`.audio-zoom-indicator`) 1.5초 후 페이드아웃

## Phase 3: 토스트 역순 쌓임

### 원인
새 토스트를 `prepend`로 DOM 맨 위에 추가하지만, z-index가 없어 DOM 순서상 나중에 추가된(오래된) 토스트가 위에 렌더링됨.

### 수정
`_updateToastStack()`에서 `t.style.zIndex = toasts.length - fromTop`으로 최신 토스트에 높은 z-index 부여.

## Phase 4: 앱 설정 모달

### 구조
- 좌측 탭 네비게이션 (개인정보, 알람, 테마) + 세로 구분선 + 우측 콘텐츠
- 600x500px, 기존 모달 디자인 패턴 활용
- 댓글 설정 드롭다운에 "앱 설정" 진입점 추가

## Phase 5: 알람 설정 탭

### 기능
- 알림 표시 on/off 토글
- 6방향 위치 선택 (좌상/상/우상/좌하/하/우하) SVG 미니 그리드
- 지속시간 슬라이더 (1~10초)
- 미리보기 박스: 선택한 위치에 샘플 토스트 표시

### 설정 저장
`user-settings.js`에 `toastPosition`, `toastDuration` 추가.
`showToast()`의 기본 duration을 `userSettings.getToastDuration()`에서 읽도록 변경.

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|--------|--------|----------|
| 줌 시 성능 저하 | 중간 | 보이는 범위만 렌더링, 리샘플링 최적화 |
| 기존 토스트 호출 duration 파라미터 | 낮음 | null 체크로 기존 하드코딩된 값 유지 |
