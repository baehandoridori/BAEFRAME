# 실시간 협업 WebSocket - 테스트 체크리스트

> Phase 0~6 구현 후 통합 테스트 (Phase 7)
> 테스트 환경: 동일 LAN 내 Windows PC 2~3대

---

## 테스트 환경 준비

- [ ] 모든 PC에서 `npm install` 완료 (`ws` 패키지 설치 확인)
- [ ] 모든 PC에서 `npm run dev`로 앱 실행 가능 확인
- [ ] 동일 Google Drive 공유 폴더에 테스트용 영상 + `.bframe` 파일 준비
- [ ] 각 PC의 사용자 이름이 다르게 설정되어 있는지 확인 (설정 > 사용자 이름)
- [ ] Windows 방화벽에서 포트 12345~12350 허용 (또는 Electron 앱 허용)

## 4. 저장 전략 (Phase 4)

### 4-1. Host-Only Write

| # | 시나리오 | 테스트 방법 | 예상 결과 | 통과 |
|---|---------|-----------|----------|------|
| 1 | Host만 파일 저장 | PC-A(Host)에서 댓글 추가 후 3초 대기 | `.bframe` 파일 modifiedAt 갱신됨 | ⬜ |
| 2 | Client는 저장 안 함 | PC-B(Client)에서 댓글 추가 후 확인 | PC-B에서 `.bframe` 직접 쓰기 없음 | ⬜ |
| 3 | Client 변경 → Host 저장 | PC-B에서 댓글 추가 → 3초 대기 | Host의 `.bframe`에 PC-B 댓글 포함 | ⬜ |

### 4-2. Debounce 저장

| # | 시나리오 | 테스트 방법 | 예상 결과 | 통과 |
|---|---------|-----------|----------|------|
| 1 | 3초 Debounce | Host에서 빠르게 댓글 3개 연속 추가 | 마지막 변경 후 3초 뒤 1회 저장 | ⬜ |
| 2 | 30초 최대 간격 | 계속 편집 중 (30초 이상) | 30초마다 강제 저장 실행 | ⬜ |

### 4-3. 종료 시 저장

| # | 시나리오 | 테스트 방법 | 예상 결과 | 통과 |
|---|---------|-----------|----------|------|
| 1 | Host 종료 시 미저장 데이터 | 편집 직후 Host 종료 | forceSave() 실행 → 데이터 보존 | ⬜ |
| 2 | 역할 전환 후 저장 | Host→Solo 전환 시 | autoSaveDelay 500ms로 복원 | ⬜ |

## 디버깅 팁

### 로그 확인
- **Main Process**: `%APPDATA%\baeframe\logs/` (또는 DevTools Console)
- **Renderer**: DevTools Console (`npm run dev`로 실행 시 자동 열림)
- **필터**: `WSServer`, `WSClient`, `CollaborationManager`, `ReviewDataManager`

### 주요 로그 메시지
```
[CollaborationManager] Host 모드로 시작 {ip, port}
[CollaborationManager] Client 모드로 시작 {hostIp, port}
[CollaborationManager] Host 후보 딜레이 {delay}
[CollaborationManager] WELCOME 수신 {participants, hasFullState}
[CollaborationManager] Host 선출 시작 {mySessionId}
[WSServer] WebSocket 서버 시작 {port, ip}
[WSClient] WebSocket 연결 성공 {url}
[WSClient] 재접속 예약 {attempt, delay}
[ReviewDataManager] 저장 역할 변경 {from, to, autoSaveDelay}
```

### 포트 확인 (Windows)
```cmd
netstat -an | findstr 12345
```

### session.json 직접 확인
```
# 영상 파일 옆에 생성됨
[영상파일명].bframe.session.json
```
