# 스플릿 뷰 오버레이 비교 모드 개발 계획

## 요약

| 항목 | 수정 파일 | 이유 | 우선순위 | 상태 |
|------|----------|------|---------|------|
| Phase 1 | index.html, split-view-manager.js | 뷰 모드 전환 인프라 + 모드 버튼 UI | 높음 | 🔄 진행중 |
| Phase 2 | split-view-manager.js, main.css | 불투명도 오버레이 모드 구현 | 높음 | ⬜ 대기 |
| Phase 3 | split-view-manager.js, main.css | 커튼 와이프 모드 구현 | 높음 | ⬜ 대기 |
| Phase 4 | split-view-manager.js | 단축키 바인딩 | 중간 | ⬜ 대기 |

## 배경 및 목적

- 애니메이션 스튜디오에서 영상 버전을 비교할 때 나란히 보기(side-by-side) 외에도 **겹쳐서 보기** 기능이 필요
- Nuke/DaVinci 등 전문 도구의 Wipe 비교 기능을 참고하되 더 직관적인 UX로 구현
- 두 가지 오버레이 모드:
  1. **불투명도 오버레이**: 두 영상을 겹쳐서 불투명도로 블렌딩
  2. **커튼 와이프**: 세로 디바이더를 드래그해서 좌/우 영상 영역 조절

## 디자인 결정사항

### 뷰 모드 체계 (기존 모드 체계 변경)
- 기존: `sync` | `independent` (재생 모드)
- 변경: **뷰 모드** (`side-by-side` | `overlay` | `wipe`) + **재생 모드** (`sync` | `independent`)
- 오버레이/와이프 모드에서는 재생 모드가 항상 `sync`로 강제됨

### 모드 버튼 UI
- 기존 "동기 재생 / 독립 재생" 버튼 → "나란히 보기 / 오버레이 / 와이프" 3개 뷰 모드 버튼으로 교체
- 나란히 보기 모드에서만 기존 sync/independent 토글 표시

### 버전 선택 UI (오버레이/와이프 모드)
- 상단 헤더에 통합: "v1 ↔ v2" 형태로 두 버전 표시
- 스왑 버튼으로 좌우 교체 가능

### 단축키
- `1`: 나란히 보기 모드
- `2`: 오버레이 모드
- `3`: 와이프 모드
- `[` / `]`: 불투명도 -10% / +10% (오버레이 모드)
- `[` / `]`: 커튼 위치 왼쪽/오른쪽 5% 이동 (와이프 모드)

## Phase 1: 뷰 모드 전환 인프라

### 목표
- `_viewMode` 프로퍼티 추가 (`'side-by-side'` | `'overlay'` | `'wipe'`)
- 헤더 버튼 UI 변경
- 모드 전환 시 DOM 레이아웃 변경 로직

### 구현 내용

#### HTML 변경 (index.html)
1. 모드 컨트롤 영역에 3개 뷰 모드 버튼 추가
2. 오버레이 모드용 불투명도 슬라이더 컨테이너 추가
3. 와이프 모드용 커튼 디바이더 추가
4. 오버레이 모드용 통합 버전 표시 바 추가

#### JS 변경 (split-view-manager.js)
1. `_viewMode` 상태 추가
2. `_setViewMode(mode)` 메서드 추가
3. 기존 `_bindEvents()`에 뷰 모드 버튼 바인딩 추가

## Phase 2: 불투명도 오버레이 모드

### 목표
- 두 비디오를 같은 공간에 겹쳐서 표시
- 상단 비디오(우측 버전)의 불투명도를 슬라이더로 조절

### 구현 내용
- CSS: 두 비디오를 absolute 포지션으로 겹침
- JS: 불투명도 슬라이더 이벤트 → 우측 비디오 opacity 변경
- 슬라이더: 영상 하단에 반투명으로 표시, 3초 비활성 시 자동 숨김

## Phase 3: 커튼 와이프 모드

### 목표
- 두 비디오를 같은 공간에 겹쳐서 표시
- 세로 디바이더 드래그로 좌/우 영상 클리핑 영역 조절

### 구현 내용
- CSS: clip-path로 좌측 비디오는 디바이더 왼쪽만, 우측 비디오는 오른쪽만 표시
- JS: 디바이더 드래그 이벤트 → clip-path 업데이트
- 디바이더: 가운데 원형 핸들 + 세로선, "좌 | 우" 버전 라벨 표시

## Phase 4: 단축키

### 구현 내용
- 기존 `_bindKeyboardEvents()`에 추가
- 모드 전환: Digit1/2/3
- 불투명도/커튼 위치: BracketLeft/Right

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|--------|--------|----------|
| 영상 겹침 시 GPU 부담 | 중간 | CSS opacity/clip-path만 사용 (GPU 가속) |
| 기존 모드 체계 변경 영향 | 중간 | _mode(sync/independent)는 유지, _viewMode만 추가 |
| 기존 캔버스 드로잉 호환 | 낮음 | 오버레이 모드에서 드로잉 캔버스는 상단 비디오에만 표시 |

## 테스트 방법

1. 나란히 보기 ↔ 오버레이 ↔ 와이프 모드 전환 확인
2. 불투명도 슬라이더 0%~100% 정상 동작 확인
3. 커튼 디바이더 드래그 부드러움 확인
4. 모드 전환 후 재생/프레임 이동 정상 동작 확인
5. 단축키 1/2/3, [ ] 정상 동작 확인
