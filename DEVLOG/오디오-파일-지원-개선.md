# 오디오 파일 지원 개선

## 요약

| 항목 | 수정 파일 | 이유 | 우선순위 | 상태 |
|------|----------|------|---------|------|
| audioWaveform 미정의 버그 | renderer/scripts/app.js | play/pause/ended 에러 방지 | 높음 | ✅ 완료 |
| MP3 웨이브폼 지원 | renderer/scripts/modules/audio-waveform.js | 비-WAV 오디오 지원 | 높음 | ✅ 완료 |
| seek 리스너 중복 등록 | renderer/scripts/app.js | 파일 변경 시 이벤트 누적 방지 | 중간 | ✅ 완료 |

## 배경 및 목적

코드 리뷰에서 오디오 관련 3건의 이슈가 발견됨:
1. `audioWaveform` 변수가 스코프에 없어 play/pause/ended 이벤트에서 ReferenceError 발생
2. 웨이브폼 생성기가 WAV만 지원하여 MP3 등 비-WAV 파일에서 에러 발생
3. `loadVideo()` 호출 시마다 seek 이벤트 리스너가 중복 등록됨

## 수정 내용

### 1. audioWaveform 미정의 버그 수정
- `audioWaveform.setPlaying()` → `getAudioWaveform()?.setPlaying()` 변경
- `getAudioWaveform()`은 이미 import된 싱글톤 getter로, 인스턴스 없으면 null 반환
- optional chaining으로 안전 호출

### 2. MP3 웨이브폼 Web Audio API 폴백 추가
- `loadAudio()`에서 IPC(WAV 전용) 시도 후 실패 시 Web Audio API 폴백
- `window.electronAPI.readBinaryFile()` → `AudioContext.decodeAudioData()` → 다운샘플링
- MP3/OGG/FLAC/AAC 등 브라우저가 지원하는 모든 오디오 포맷 자동 지원

### 3. seek 리스너 중복 등록 수정
- 핸들러를 `audioWaveform._seekHandler`에 저장
- 새 리스너 등록 전 이전 핸들러 제거
