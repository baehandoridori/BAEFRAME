# 재생 동기화 기능 구현 계획

## 요약

| 항목 | 수정 파일 | 이유 | 우선순위 | 상태 |
|------|----------|------|---------|------|
| Phase 1 | `renderer/scripts/modules/playback-sync.js` (신규) | 동기화 핵심 로직 모듈 | 높음 | ✅ 완료 |
| Phase 2 | `renderer/index.html`, `renderer/styles/main.css` | 플로팅 패널 UI | 높음 | ✅ 완료 |
| Phase 3 | `renderer/scripts/app.js` | 모듈 통합 및 이벤트 연결 | 높음 | ✅ 완료 |

## 배경 및 목적

- 동일 세션(Liveblocks Room)에 접속한 사용자들의 재생 상태를 동기화
- 한 사용자가 재생/일시정지/탐색하면 다른 사용자도 동일하게 반영
- 기존 Liveblocks Broadcast 인프라를 활용하여 구현

## 아키텍처

```
사용자 A (재생 클릭)
  → PlaybackSync.broadcastPlay()
  → liveblocksManager.broadcastEvent({ type: 'PLAYBACK_PLAY', time, ... })
  → WebSocket → 모든 Room 멤버에게 전달

사용자 B (수신)
  → PlaybackSync._onBroadcast()
  → type === 'PLAYBACK_PLAY' → videoPlayer.play() + seek(time)
```

### 브로드캐스트 이벤트 타입

| 이벤트 | 데이터 | 설명 |
|--------|--------|------|
| `PLAYBACK_PLAY` | `{ time }` | 재생 시작 |
| `PLAYBACK_PAUSE` | `{ time }` | 일시정지 |
| `PLAYBACK_SEEK` | `{ time }` | 위치 이동 |
| `PLAYBACK_SYNC_MODE` | `{ mode }` | 모드 변경 (anyone/host) |

### 리더 모드

1. **anyone (기본)**: 누구든 조작하면 전원 따라감
2. **host**: 방을 만든 사용자(또는 지정된 호스트)만 주도 가능

## Phase 1: playback-sync.js 모듈

### 목표
- Liveblocks Broadcast를 통한 재생 상태 동기화 핵심 로직

### 구현 내용
- `PlaybackSync` 클래스 (EventTarget 상속)
- `start(liveblocksManager)` / `stop()` 메서드
- 브로드캐스트 송신: `broadcastPlay()`, `broadcastPause()`, `broadcastSeek(time)`
- 브로드캐스트 수신: `_onBroadcast(event)` → 이벤트 디스패치
- 동기화 on/off: `setSyncEnabled(boolean)`
- 리더 모드: `setLeaderMode('anyone' | 'host')`
- 루프 방지: `isRemoteUpdate` 플래그
- 탐색 쓰로틀링: 100ms (빠른 드래그 시 과도한 브로드캐스트 방지)

## Phase 2: 플로팅 패널 UI

### 목표
- 동기화 상태를 보여주고 제어하는 플로팅 패널

### UI 요소
- 동기화 토글 체크박스 (on/off)
- 리더 모드 선택 (누구나 / 호스트만)
- 접속자 수 표시
- 동기화 상태 표시 (동기화 중 / 끊김)
- 패널 접기/펼치기

## Phase 3: app.js 통합

### 목표
- PlaybackSync 모듈과 기존 재생 이벤트 연결

### 구현 내용
- play/pause/ended/seek 이벤트에서 broadcastPlay/Pause/Seek 호출
- PlaybackSync에서 발생하는 remotePlay/remotePause/remoteSeek 이벤트 수신 → videoPlayer 제어
- 플로팅 패널 초기화 및 이벤트 바인딩

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|--------|--------|----------|
| 무한 루프 (A→B→A) | 높음 | isRemoteUpdate 플래그로 차단 |
| 네트워크 지연 | 중간 | seek 시 시간 보정, 100ms 이하 차이 무시 |
| 과도한 브로드캐스트 | 중간 | seek 쓰로틀링 100ms |
| Liveblocks 미연결 시 | 낮음 | 연결 상태 확인 후 브로드캐스트 |

## 테스트 방법
1. 두 인스턴스에서 같은 .bframe 파일 열기
2. 한쪽에서 재생/정지/탐색 → 다른쪽 반영 확인
3. 동기화 체크박스 해제 → 독립 재생 확인
4. 호스트 모드 전환 → 비호스트 사용자 조작 무시 확인
