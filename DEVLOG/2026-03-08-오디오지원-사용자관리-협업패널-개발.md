## 배경 및 목적

1. **오디오 파일 지원**: 애니메이션 스튜디오에서 음성 녹음(mp3/wav) 리뷰 필요
2. **초기비밀번호 표기**: 관리자가 비밀번호 미변경 사용자를 빠르게 파악
3. **협업 플렉서스 패널**: 연결된 사용자를 시각적으로 확인하는 인터랙션

## Phase 1: 오디오 확장자 등록

### 수정 파일
- `main/ipc-handlers.js` - 파일 다이얼로그 필터
- `renderer/scripts/app.js` - SUPPORTED_EXTENSIONS, isVideoFile → isMediaFile
- `shared/schema.js` - 확장자 목록
- `shared/playlist-schema.js` - 재생목록 확장자

## Phase 2: AudioWaveform 모듈

### 신규 파일: `renderer/scripts/modules/audio-waveform.js`
- Web Audio API로 오디오 데이터 디코딩
- Canvas로 실시간 웨이브폼 렌더링
- 스크러빙 (클릭/드래그로 탐색)
- 재생 헤드 동기화
- 그라데이션 + 미러 웨이브폼 디자인

## Phase 3: 오디오 모드 UI

### 수정 내용
- `loadVideo` → 오디오 감지 시 오디오 모드 활성화
- 비디오 영역을 웨이브폼 캔버스로 대체
- 타임라인/재생 컨트롤은 그대로 유지
- 그리기 모드 비활성화

## Phase 4: 초기비밀번호 표기

### 수정 내용
- `auth-manager.js`: `hasInitialPassword()` 메서드 추가
- `app.js`: `renderRegisteredUsers()`에 배지 표시
- `main.css`: 배지 스타일

## Phase 5: 협업 플렉서스 패널

### 수정 내용
- `index.html`: 확장 패널 HTML
- `main.css`: 호버 확장 애니메이션
- `app.js`: 호버 이벤트 + PlexusEffect 연동
- `user-settings.js`: 토글 설정 추가

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|--------|--------|----------|
| Web Audio API 대용량 파일 | 중간 | 청크 기반 디코딩, 다운샘플링 |
| 기존 비디오 기능 호환성 | 높음 | isAudioMode 플래그로 분기 |
| 플렉서스 성능 | 낮음 | 기존 검증된 PlexusEffect 재사용 |

## 테스트 방법
1. mp3/wav 파일 드래그앤드롭으로 열기
2. 웨이브폼에서 클릭/드래그로 탐색
3. 사용자 관리에서 초기비밀번호 배지 확인
4. 협업 세션 진입 후 헤더 호버로 패널 확인
