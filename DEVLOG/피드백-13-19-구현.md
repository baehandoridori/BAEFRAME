## 배경 및 목적

- 드로잉 작업 중 레이어 생성 위치, 화면 이동, 펜 입력, 선택/복사 동작이 Animate류 작업 습관과 다르게 동작하던 문제를 정리했다.
- 댓글 수정 중 @멘션이 끊기는 흐름도 같이 보완했다.
- `.bframe` 저장 포맷은 변경하지 않고, 선택/이동 결과는 기존 키프레임 비트맵으로 저장되도록 구현했다.

## Phase 1: 댓글 수정 멘션

### 목표
- 댓글을 새로 쓸 때뿐 아니라 수정할 때도 @멘션 검색과 선택이 동작하게 한다.

### 수정 파일
- `renderer/scripts/app.js`

### 구현 내용
- 댓글 수정 textarea에 mention manager attach/detach를 연결했다.
- 멘션 드롭다운이 열린 상태에서는 Enter 키가 댓글 저장보다 멘션 선택에 우선하도록 막았다.

### 예상 리스크
- 댓글 저장 Enter 동작과 멘션 선택 Enter 동작이 겹칠 수 있어, 멘션 드롭다운 표시 여부를 먼저 확인하도록 했다.

## Phase 2: 레이어와 단축키

### 목표
- 새 드로잉 레이어를 현재 레이어 위 행에 추가한다.
- Animate식 단축키를 추가한다.

### 수정 파일
- `renderer/scripts/app.js`
- `renderer/scripts/modules/user-settings.js`
- `renderer/scripts/modules/drawing-manager.js`
- `renderer/scripts/modules/drawing-sync.js`
- `scripts/tests/keyframe-shortcuts-source.test.js`
- `scripts/tests/playlist-autoplay-source.test.js`

### 구현 내용
- `DrawingManager.createLayer()`에 `insertIndex`를 추가하고, 협업 이벤트에도 같은 값을 실었다.
- `Shift+V`, `` ` ``, `Ctrl+2`, `Ctrl+Alt+C`, `Ctrl+Alt+V` 단축키를 추가했다.
- 프레임 복사/붙여넣기는 드로잉 매니저의 키프레임 복사 API로 묶었다.

### 예상 리스크
- 기존 클라이언트와 협업할 때 위치 정보가 없는 레이어 생성 이벤트가 올 수 있어, `insertIndex`가 없으면 기존처럼 끝에 추가되도록 fallback을 유지했다.

## Phase 3: 스페이스 홀드 패닝

### 목표
- 드로잉 모드에서는 Space를 눌렀다 떼면 재생, 누른 채 드래그하면 화면 패닝으로 동작하게 한다.

### 수정 파일
- `renderer/scripts/app.js`
- `renderer/styles/main.css`
- `scripts/tests/keyboard-shortcut-targets.test.js`

### 구현 내용
- `state.isSpaceHeld`, `state.spacePanUsed`를 추가했다.
- Space 홀드 중에는 활성 드로잉 오버레이의 포인터 이벤트를 잠시 비워서 비디오 패닝이 먼저 잡히게 했다.
- 드래그가 있었던 Space keyup은 재생 토글로 처리하지 않게 분기했다.

### 예상 리스크
- 일반 재생 단축키 회귀를 막기 위해 드로잉 모드 밖에서는 기존 keydown 재생 흐름을 유지했다.

## Phase 4: 펜 입력과 도형 외곽선

### 목표
- 디지타이저/펜 입력이 캔버스와 타임라인에서 끊기지 않게 한다.
- 선/도형 도구도 브러시 외곽선 설정을 반영한다.

### 수정 파일
- `renderer/scripts/modules/drawing-canvas.js`
- `renderer/scripts/modules/timeline.js`
- `renderer/styles/main.css`
- `scripts/tests/brush-settings-source.test.js`
- `scripts/tests/keyframe-multi-select-source.test.js`

### 구현 내용
- 드로잉 캔버스 입력을 Pointer Events 기반으로 전환하고 `setPointerCapture`를 사용했다.
- 타임라인 스크럽/마커 조작도 `pointerdown`, `pointermove`, `pointerup`, `pointercancel` 흐름으로 맞췄다.
- 도형 렌더링은 외곽선과 본선을 분리해서 그리도록 정리했다.

### 예상 리스크
- 마우스 입력도 Pointer Events를 타기 때문에, 기존 마우스 동작이 유지되는지 관련 소스 회귀 테스트를 갱신했다.

## Phase 5: 선택 도구

### 목표
- 드로잉 일부를 선택해서 이동하고, `Ctrl+C`/`Ctrl+V`로 복사/붙여넣기할 수 있게 한다.

### 수정 파일
- `renderer/index.html`
- `renderer/styles/main.css`
- `renderer/scripts/app.js`
- `renderer/scripts/modules/user-settings.js`
- `renderer/scripts/modules/drawing-canvas.js`
- `renderer/scripts/modules/drawing-manager.js`
- `scripts/tests/drawing-runtime-source.test.js`

### 구현 내용
- 선택 전용 오버레이 캔버스와 select 도구 버튼을 추가했다.
- 마퀴 선택, 선택 영역 들어올리기, 이동, 커밋, 내부 클립보드 복사/붙여넣기 흐름을 추가했다.
- 도구 전환, 드로잉 모드 종료, Escape, 프레임 변경 시 플로팅 선택 영역이 먼저 커밋되도록 했다.

### 예상 리스크
- 플로팅 선택 영역이 프레임 이동 중 유실될 수 있어 `setCurrentFrame()`과 `renderFrame()` 진입 전에 커밋 가드를 넣었다.

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|--------|--------|----------|
| 전체 `npm run lint`가 기존 CRLF/레거시 린트 항목에 막힘 | 중간 | 변경 JS 파일 대상 `npx eslint`를 별도로 실행했고 에러 0개를 확인 |
| 선택 도구 결과가 스트로크 레코드와 섞임 | 중간 | 선택 이동/붙여넣기 결과는 도형 도구와 같이 비트맵으로 확정하고 stroke records를 freeze 처리 |
| 기존 하이라이트 복사/붙여넣기와 단축키 충돌 | 낮음 | `Ctrl+Alt+C/V`와 select 도구 `Ctrl+C/V` 조건을 분리하고 capture 단계에서 소비 |
