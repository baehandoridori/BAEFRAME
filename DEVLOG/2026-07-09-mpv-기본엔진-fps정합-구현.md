# mpv 기본 엔진 승격 + fps 정합 구현

> **날짜**: 2026-07-09
> **상태**: ✅ 구현 완료
> **계획**: `docs/superpowers/plans/2026-07-09-mpv-default-engine-fps-accuracy.md`
> **사전 검수**: `DEVLOG/2026-07-09-mpv-통합-심층검수-보고서.md`

---

## 요약

BAEFRAME의 기본 영상 재생 경로를 mpv 직접 재생으로 승격하고, 프레임 표기와 1프레임 이동이 영상의 실제 fps를 기준으로 동작하도록 정리했다. 기존 FFmpeg 변환 경로는 삭제하지 않고, mpv가 없거나 실패하거나 사용자가 끈 경우 자동으로 기존 방식으로 돌아가게 유지했다.

---

## 진행 단계

| Phase | 내용 | 상태 |
|---|---|---|
| Chunk 1 | ffprobe/mpv/HTML5 fps 값을 소수 fps까지 보존하고, 타임코드와 프레임 표기를 실제 fps 기준으로 보정 | ✅ |
| Chunk 1 | 과거 24fps 고정 시절에 저장된 `.bframe` 댓글/드로잉 프레임을 로드 시 현재 fps 기준으로 재매핑 | ✅ |
| Chunk 2 | mpv 응답 없음, 예기치 않은 종료, 렌더러 reload/crash 시 자동 복구 및 정리 경로 추가 | ✅ |
| Chunk 3 | mpv 모드 그리기 진입 시 현재 프레임 PNG 스크린샷을 캔버스 아래 정지 프레임으로 표시 | ✅ |
| Chunk 4 | `mpvPlaybackEnabled` 기본값 true 전환, 구 `mpvPilotEnabled` 저장값 마이그레이션, `BAEFRAME_DISABLE_MPV=1` opt-out 추가 | ✅ |

## 주요 변경

- `main/ffmpeg-manager.js`: `24000/1001` 같은 분수 fps를 `23.976`처럼 소수점 3자리까지 유지.
- `renderer/scripts/modules/video-player.js`: HTML5/mpv 전환 시 fps 잔존값을 정리하고, `setFps()`가 메타데이터를 다시 전파하게 보강.
- `renderer/scripts/app.js`: HTML5 로드 경로에도 ffprobe fps를 주입하고, mpv 중단 시 1회 자동 재시도 후 기존 경로로 폴백.
- `renderer/scripts/modules/comment-manager.js`, `drawing-manager.js`, `review-data-manager.js`: 저장/로드 시 fps 정합성을 맞추고 과거 프레임 데이터를 보정.
- `main/mpv-manager.js`, `main/ipc-handlers.js`, `preload/preload.js`: mpv 스크린샷 IPC와 환경변수 opt-out 추가.
- `renderer/index.html`, `renderer/scripts/modules/user-settings.js`: mpv 직접 재생을 기본 설정으로 전환하고 기존 파일럿 키를 폐기.
- `web-viewer/scripts/app.js`: `.bframe` 최상위 `fps` 필드를 fallback 체인에 포함.

## 리스크와 운영 메모

- mpv 직접 재생이 기본값이므로 팀원 전원에게 즉시 영향이 있다. 문제가 있는 PC는 설정의 mpv 직접 재생 토글을 끄거나 `BAEFRAME_DISABLE_MPV=1`로 우회한다.
- mpv 모드의 타임라인 필름스트립, 스크럽 프리뷰, 댓글 썸네일은 아직 기존처럼 비활성 상태다. 이번 작업의 `mpv:screenshot` IPC가 후속 구현 기반이다.
- mpv 상태 동기화는 아직 120ms 폴링 기반이라 재생 중 프레임별 드로잉 갱신은 일부 건너뛸 수 있다.
- 스플릿 뷰 자체 fps 기본값 24 고정은 이번 범위 밖이다.
- 웹뷰어 fps fallback이 바뀌었으므로 웹뷰어 배포 환경은 PR 병합 후 재배포가 필요하다.
- 구버전과 신버전 사용자가 같은 협업 세션에 섞이면 fps 해석 차이로 재생헤드/댓글 위치가 어긋날 수 있다. 팀 전원 동시 업데이트를 권장한다.

### 수동 검증

1. 24fps, 30fps, 23.976fps 영상을 각각 열어 프레임 카운터와 1프레임 이동이 실제 fps 기준으로 동작하는지 확인.
2. 각 샘플에서 댓글 작성, 저장, 재로드 후 같은 프레임에 표시되는지 확인.
3. 기존 24fps `.bframe` 파일을 열어 댓글/드로잉 위치가 변하지 않는지 확인.
4. mpv 설정 on/off 각각에서 영상 로드 경로가 mpv 직접 재생과 기존 FFmpeg/HTML5 경로로 바뀌는지 확인.
5. `BAEFRAME_DISABLE_MPV=1` 상태에서 mpv가 설치되어 있어도 기존 경로로 재생되는지 확인.
6. 빌드된 `dist/win-unpacked/BFRAME_alpha_v2.exe`에서 mpv 기본 재생, 그리기 정지 프레임, mpv 강제 종료 후 복구를 스모크 테스트.
