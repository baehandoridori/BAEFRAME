## 배경 및 목적

- 피드백 13~19 구현 이후에도 사용자 화면 기준 레이어 위치, 단순 재생 시 리뷰 파일 생성, 펜 Alt 입력, mpv 정지 seek에서 실제 작업 흐름을 방해하는 문제가 남아 있었다.
- 피드백 23은 합성 순서보다 레이어 패널에서 보이는 위/아래 방향을 사용자 기준으로 맞추는 작업이다.
- 피드백 20은 댓글·드로잉 등 실제 리뷰 데이터가 없는 영상을 재생했을 뿐인데 `.bframe` 파일이 생겨 파일 관리가 복잡해지는 문제를 막는다.
- 피드백 22는 일부 펜 드라이버가 포인터 이벤트에 `altKey`를 전달하지 않아 두께 조절 중 획이 그려질 수 있는 입력 차이를 방어한다.
- 피드백 24는 정지 상태에서 프레임을 옮긴 뒤 mpv의 미세한 시간 오차가 목표 프레임을 N-1로 덮어쓰지 않게 한다.
- mpv 재생 중 requestAnimationFrame(rAF) 보간은 `feat: mpv 재생 중 프레임 보간 작업 보존`이라는 **별도 선행 커밋**으로 보존했다. 전체 리뷰에서 늦은 상태 폴링이 재생 프레임을 뒤로 당길 수 있는 문제를 발견해, 보간은 전진만 하도록 같은 선행 커밋에 동적 회귀 테스트와 함께 보강했다. 피드백 24 커밋에는 보간 필드·함수·rAF 루프를 새로 추가하거나 재작성한 변경이 섞이지 않았다.

## Phase 1: 피드백 23 — 레이어를 패널 기준 위 행에 추가

### 목표

- Shift+F1 또는 `+` 버튼으로 새 레이어를 만들면 현재 활성 레이어의 패널 바로 위 행에 나타나게 한다.

### 수정 파일

- `renderer/scripts/app.js`
- `scripts/tests/keyframe-shortcuts-source.test.js`

### 구현 내용

- `addDrawingLayer()`에서 활성 레이어가 있을 때 삽입 위치를 `activeIndex + 1`이 아니라 `activeIndex`로 변경했다.
- 활성 레이어가 없으면 기존처럼 레이어 배열 끝에 추가한다.
- `drawingManager.createLayer({ insertIndex, insertBeforeLayerId })` 경로를 사용해 undo와 활성화는 유지하면서, 협업 이벤트에는 숫자 위치뿐 아니라 기준 레이어 ID도 전달한다.
- 새 레이어 ID를 Web Crypto 기반 UUID로 만들어 두 사용자가 동시에 같은 `layer-N` ID를 만드는 충돌을 제거했다.
- 같은 기준 레이어 앞에서 동시에 생성된 레이어는 ID 기준으로 결정론적으로 정렬한다. 기준 레이어 이벤트가 늦거나 먼저 삭제된 경우, 수동 레이어 이동, create/delete/order undo·redo도 실행형 두 피어 테스트로 고정했다.
- Undo/Redo는 작업별 lifecycle metadata로 정확한 대상만 복원·삭제·이동한다. 스냅샷 이후 상대가 추가한 레이어와 비대상 레이어의 객체·최신 키프레임은 그대로 보존한다.
- 동시 수동 이동은 Lamport 시각과 사용자 ID tie-break로 모든 피어가 같은 순서를 선택한다. 삭제 Undo 복원은 새 복원 이벤트와 함께 구버전용 생성·키프레임 이벤트도 보내 현재 배포본과의 혼합 세션을 지원한다.
- 1MB를 넘는 키프레임은 전체 payload를 제한 이하 청크로 나눠 재조립하며, 동시 전송 수·개별/전체 버퍼 크기·타임아웃 상한으로 손상되거나 악의적인 전송의 메모리 사용을 제한한다.
- 새 참여자에게 보내는 현재 상태도 같은 제한 경로를 사용한다. 구버전 혼합 세션에는 제한 이하 축소본을 먼저 보내고 새 버전에는 원본 청크로 덮어쓴다. 축소가 불가능한 1MB 초과 원본의 정확한 복원은 모든 협업자가 이번 버전으로 재시작한 세션에서 보장한다.
- 기준점과 삭제 tombstone은 런타임/undo 스냅샷에만 보관하며 `.bframe` 저장 스키마에는 추가하지 않았다. 구버전 `insertIndex` 이벤트도 계속 처리한다.
- 소스 회귀 테스트가 패널 방향 주석과 새 삽입 인덱스를 확인하고, 동적 테스트가 실제 `DrawingManager`와 `DrawingSync` 두 피어의 수렴을 검증한다.
- 구현 커밋: `fix: Shift+F1 레이어를 패널 기준 위쪽에 추가 (피드백 23)`

### 예상 리스크

- 패널 배열의 인덱스 0이 최상단인 구조를 우선했기 때문에, 새 레이어는 합성 기준으로는 기존 활성 레이어 뒤에 놓인다. 이는 사용자에게 보이는 패널 방향을 우선한 의도된 트레이드오프다.
- 동시 생성·이벤트 지연·anchor 삭제·수동 이동·create/delete/order undo·redo는 자동 검증했지만, 실제 Liveblocks 두 창을 연결한 수동 검증은 아직 남아 있다.

## Phase 2: 피드백 20 — 단순 재생 시 `.bframe` 생성 차단

### 목표

- 새 리뷰 파일이 없는 영상을 열고 재생만 했을 때 FPS 같은 자동 메타데이터 변경만으로 `.bframe` 파일이 생성되지 않게 한다.
- 기존 리뷰 파일, 실제 리뷰 데이터, 명시적 수동 저장 동작은 유지한다.

### 수정 파일

- `renderer/scripts/modules/review-data-manager.js`
- `scripts/tests/lazy-bframe-creation-source.test.js`

### 구현 내용

- `hasSubstantiveContent()`를 추가해 댓글, 드로잉 키프레임, 하이라이트, 합성 레이어, 수동 버전의 존재 여부를 판정한다.
- `hasUnsavedChanges()`가 dirty 여부뿐 아니라 기존 파일 존재 여부와 실제 리뷰 내용도 함께 확인하도록 변경했다.
- 영상 전환 전 저장과 자동 저장 타이머가 raw `isDirty` 대신 `hasUnsavedChanges()`를 사용하게 했다.
- 기존 `.bframe` 파일이 있으면 메타데이터 변경도 계속 저장한다. 사용자가 직접 실행한 저장과 링크 복사의 `save()` 경로도 변경하지 않았다.
- `.bframe` 스키마, 직렬화 포맷, 협업 discovery/join/merge 로직은 수정하지 않았다.
- 실제 `ReviewDataManager`를 실행하는 동적 테스트를 추가해 metadata-only 영상 전환·자동저장은 저장 0회, 댓글 추가 뒤 첫 저장은 1회, 기존 파일의 metadata dirty는 저장 대상으로 유지되는지 확인한다.
- PR 1차 리뷰에서 소프트 삭제된 댓글만 남은 경우도 실질 데이터가 아닌 것으로 판정하도록 보강했다.
- 구현 커밋: `fix: 실질 데이터 없는 단순 재생 시 .bframe 파일 생성 차단 (피드백 20)`

### 예상 리스크

- 빈 드로잉 레이어만 만들고 키프레임을 추가하지 않은 상태는 실질 데이터로 보지 않아 새 `.bframe`을 만들지 않는다. 계획에 따른 동작이다.
- 삭제된 댓글 마커는 협업 상태 보존을 위해 Map에 남으므로 실질 데이터로 판정되어 저장된다.
- 자동 테스트는 저장 조건의 소스 계약을 확인하지만, Electron 앱에서 실제 파일이 생성되지 않는지 확인하는 수동 검증은 아직 남아 있다.

## Phase 3: 피드백 22 — Alt+펜 두께 조절 하드닝

### 목표

- 펜 드라이버가 포인터 이벤트의 `altKey`를 비워 보내더라도 Alt+펜 드래그가 획 그리기가 아니라 브러시 두께 조절로 동작하게 한다.

### 수정 파일

- `renderer/scripts/modules/drawing-canvas.js`
- `scripts/tests/brush-settings-source.test.js`

### 구현 내용

- 생성자에 `_modifierAltDown` 상태를 추가했다.
- document의 Alt keydown/keyup과 window blur를 추적해 실제 키보드 Alt 상태를 보관하고 포커스를 잃으면 초기화한다.
- `_isAltActive(e)`를 추가해 이벤트의 `altKey`와 전역 Alt 상태를 함께 판정한다.
- `_onPointerDown()`과 `_onMouseDown()`의 Alt 조건을 이 헬퍼로 통일했다. 두께 조절을 시작한 뒤 즉시 반환하는 기존 조기 종료 흐름은 유지했다.
- 소스 회귀 테스트가 Alt 상태 초기화, keydown/keyup/blur, 헬퍼 조건, 포인터 가드, 마우스 조기 종료를 직접 확인하도록 보강했다.
- 구현과 리뷰 테스트 보강은 `fix: 펜 입력 Alt 키 전역 보정으로 두께 조절 중 오드로잉 방지 (피드백 22)` 한 커밋에 정리했다.

### 예상 리스크

- 실제 디지타이저의 modifier 전달 방식은 펜 드라이버마다 다를 수 있어 소스 테스트만으로 모든 환경을 재현할 수 없다.
- 마우스는 기존 `e.altKey` 결과를 그대로 사용하고, Alt+Tab 등으로 포커스를 잃으면 전역 상태를 해제해 Alt가 눌린 채로 고착되는 것을 방지한다.
- 실제 펜으로 HUD만 표시되고 획이 남지 않는지 확인하는 수동 검증은 아직 남아 있다.

## Phase 4: 피드백 24 — 정지 seek 후 프레임 -1 되돌림 방지

### 목표

- A/D, Shift+A/D, 타임라인 스크럽으로 이동한 목표 프레임이 mpv 상태 폴링 후에도 유지되게 한다.
- 재생 시작, 실제 외부 프레임 변화, 엔진 종료 시에는 목표 프레임 유지 상태를 안전하게 해제한다.

### 수정 파일

- `renderer/scripts/modules/video-player.js`
- `scripts/tests/mpv-runtime-source.test.js`
- `scripts/tests/playhead-frame-step-source.test.js`

### 구현 내용

- `_timeToFrame()`의 경계 허용치를 `1e-4`에서 `0.05`로 조정해 컨테이너 PTS 양자화와 소수 FPS 절삭으로 N-1이 되는 오차를 흡수했다.
- 시간 seek와 프레임 seek의 목표를 `_pausedSeekHoldFrame`에 저장한다.
- 정지 상태의 mpv 폴링 프레임이 목표의 ±1 이내면 양자화 오차로 보고 목표 프레임과 그 시간을 유지한다.
- 폴링 값이 목표에서 2프레임 이상 다르거나 재생이 시작되거나 unload가 실행되면 hold를 해제하고 실제 상태를 반영한다.
- 소스 회귀 테스트가 초기화, 두 seek 진입점, 재생 시작, 정지 폴링 분기, 큰 차이 fallback, unload의 상태 전이 계약을 확인하도록 갱신·보강됐다.
- 실제 `VideoPlayer` 메서드를 실행하는 동적 테스트가 정지 seek의 N-1 유지/N-2 채택과 재생 중 늦은 poll에서도 프레임이 역행하지 않는지 확인한다.
- 구현과 리뷰 테스트 보강은 `fix: 정지 seek 후 mpv 폴링이 프레임바를 -1 되돌리는 문제 수정 (피드백 24)` 한 커밋에 정리했다.
- 선행 rAF 보간 커밋은 이 Phase의 구현이 아니다. 피드백 24는 기존 보간 코드를 전제로 정지 상태 seek 보호만 추가했다.

### 예상 리스크

- 정지 상태에서 mpv가 실제로 목표에서 1프레임 어긋난 경우도 양자화 오차로 간주한다. 대신 2프레임 이상 차이, 재생 시작, unload에서는 즉시 hold를 버려 외부 플레이어 상태로 복귀한다.
- 자동 테스트는 실제 객체 상태 전이와 보조 소스 계약 검사를 함께 수행하지만, 실제 mpv 영상과 HTML5 영상에서 프레임 이동·재생 전환·구간 반복을 조작하는 수동 검증은 아직 남아 있다.

| 동시 레이어 생성의 ID·순서 충돌 | 중간 | UUID ID, anchor 이벤트, 결정론 정렬, 삭제 tombstone 및 두 피어 실행 테스트로 수렴 보장 |
| 순서 이벤트가 레이어 생성보다 먼저 도착함 | 중간 | 누락 레이어가 생성되거나 삭제된 것으로 확인될 때까지 최신 순서 이벤트를 보류한 뒤 자동 재적용 |
| 일부 레이어 정렬이 대기 중인 anchor를 지움 | 중간 | 정렬 이벤트에 포함된 레이어의 anchor만 해제하고, 생성을 기다리는 생략 레이어 anchor는 보존 |

### 수동 검증 남음

- 레이어: 여러 레이어 중간을 선택하고 Shift+F1 및 `+`를 각각 실행해 패널 바로 위 행에 추가되는지 확인한다. 가능하면 협업 두 창에서도 같은 순서인지 확인한다.
- 저장: 기존 `.bframe`이 없는 영상을 열어 재생·정지·seek 후 영상 전환과 앱 종료를 수행하고 파일이 생기지 않는지 확인한다. 댓글 또는 드로잉 키프레임을 만든 뒤에는 파일이 생성되는지 확인한다.
- 펜: 실제 디지타이저로 Alt+펜 드래그 시 크기 HUD만 변하고 획이 그려지지 않는지 확인한다. Alt를 놓은 뒤 일반 펜 드로잉과 마우스 Alt 드래그도 확인한다.
- mpv: 정지 상태에서 A/D, Shift+A/D, 타임라인 스크럽 후 0.3초 이상 기다려도 목표 프레임이 -1 되돌아가지 않는지 확인한다. 재생/일시정지, 재생 중 프레임 증가, 구간 반복도 확인한다.
- HTML5: mpv 재생을 끄고 같은 프레임 이동, 재생/일시정지, 구간 반복을 확인해 기존 동작의 회귀가 없는지 확인한다.

> 자동 검증은 모두 완료 상태다. 다만 실제 펜 장치, 실제 mpv 영상, HTML5 영상으로 수행하는 수동 검증은 아직 완료되지 않았다.
