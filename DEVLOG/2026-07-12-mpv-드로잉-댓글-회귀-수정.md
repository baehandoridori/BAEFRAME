## 배경 및 목적

mpv가 기본 재생 엔진이 된 뒤 다음 증상이 보고됐다.

- 그리기/댓글 입력 중 native 영상이 사라져 검은 배경만 남는다.
- 드로잉과 댓글 마커가 mpv 영상 위에 보이지 않는다.
- `C`를 누르고 영상을 클릭해도 marker 생성이 안정적으로 시작되지 않는다.

병렬 코드 조사, 실제 배포 앱 재현, `%APPDATA%\baeframe\logs` 확인 결과 두 계층의 문제가 겹쳤다.

1. mpv overlay BrowserWindow의 cooked inline script가 invalid regex로 parse되지 않아 모든 mirrored overlay update가 실패한다.
2. 댓글/그리기 mode는 원본 DOM 입력을 위해 native host를 숨기지만 base frame이 decode되기 전에 숨기거나 댓글에는 freeze가 없어 검은 화면이 노출된다.

## Phase 1: overlay 문서 실행 계약

### 목표

- actual cooked HTML의 inline script가 compile되는지 테스트한다.
- overlay API 전역 함수가 준비된 경우에만 host 준비 성공으로 인정한다.
- readiness 전 state update를 보내지 않는다.

### 확인된 증거

- `/expression\s*\(/`가 template literal에서 cooked될 때 `/expressions*(/`로 바뀌며 `Unterminated group` 오류가 난다.
- 실제 배포 로그에서 `mpv overlay state update failed`가 수천 회 반복되며, overlay가 준비되지 않은 상태에서도 고해상도 PNG 동기화를 계속 시도했다.

### 적용 내용

- overlay HTML을 `String.raw`로 생성해 정규식과 이스케이프가 실제 브라우저 문서에서도 보존되게 했다.
- 테스트가 소스 조각이 아니라 실제 생성된 inline script 전체를 컴파일하도록 바꿨다.
- drawing overlay와 mirrored overlay API가 모두 생성된 뒤에만 준비 완료를 반환한다.

## Phase 2: mpv fail-closed

### 목표

- overlay host가 준비되지 않으면 mpv를 채택하지 않고 기존 HTML5 fallback으로 이어간다.
- renderer가 실패 결과를 무시한 채 full-resolution canvas PNG를 계속 만들지 않게 한다.

### 적용 내용

- overlay 준비 실패 시 mpv adoption을 중단하고 HTML5 재생으로 안전하게 되돌린다.
- 동일 세션의 반복 overlay 전송을 차단하는 circuit breaker와 1회 경고를 추가했다.
- 영상 전환 중 이전 준비 응답이 새 영상 상태를 덮지 못하도록 overlay generation 소유권을 검증한다.
- 정지 처리와 새 영상 로드를 직렬화하고, 새 load 소유권 획득을 원자적으로 만들어 이전 stop이 새 mpv 세션을 종료하지 못하게 했다.

## Phase 3: 댓글·그리기 review surface

### 목표

- screenshot data URL을 별도 Image에서 decode한 뒤에만 native host를 숨긴다.
- `C` mode도 같은 freeze surface를 사용해 원본 marker DOM이 클릭 가능하게 한다.
- mode 종료 시 native host가 복구된 뒤 freeze image를 제거한다.
- draw playback screenshot refresh를 마지막 요청까지 보존하는 coalescing 방식으로 바꾼다.

### 적용 내용

- 댓글과 그리기가 같은 decoded freeze surface와 소유권을 공유하도록 통합했다.
- 스크린샷을 별도 `Image`에서 decode하고, native mpv host 숨김 성공 응답까지 받은 뒤에만 marker 클릭과 그리기 입력을 연다.
- 준비 실패 시 native 영상을 계속 표시한 채 입력 모드를 종료해 검은 화면에 갇히지 않게 했다.
- 모드 종료 시 native host 복원을 먼저 확인한 뒤 freeze 이미지를 제거한다.
- 직접 캡처와 예약 캡처가 하나의 in-flight 작업을 공유하고 마지막 갱신 요청을 보존하게 해, 재생 중에도 영상 프레임과 드로잉이 함께 갱신되도록 했다.
- 댓글과 그리기 모드가 빠르게 겹쳐도 한 모드의 종료가 다른 모드의 freeze를 조기에 해제하지 않게 했다.
- 댓글과 그리기 진입을 중앙 상태 처리에서 상호 배제해 단축키·버튼·사이드바 입력이 같은 안전 경로를 사용하게 했다.
- 파일·프레임·세대가 일치하는 캡처만 입력을 다시 활성화하고, 오래된 공유 캡처가 끝난 뒤에는 현재 프레임을 한 번 더 보장 캡처하게 했다.
- `B`로 그리기를 종료할 때 마지막 드로잉 상태를 mpv 오버레이에 강제 동기화해 그리기 모드 밖에서도 그림이 계속 보이게 했다.
- mpv 중지·HTML5 전환 응답이 늦게 도착해 새 재생 엔진을 덮지 못하도록 engine/control 소유권을 다시 확인한다.
- 영상 전환이 리뷰 데이터 초기화 이후 실패하면 댓글·그리기 입력과 보존 화면, mpv 소유권을 안전하게 종료해 빈 상태가 이전 `.bframe`에 저장될 가능성을 차단했다.
- 새 영상 로드 중 잠시 보류된 오버레이 fallback은 로드 종료 후 현재 소유권을 다시 확인해 재시도한다.

## 리스크 및 우려 사항

| 리스크 | 심각도 | 완화 방안 |
|---|---|---|
| screenshot/decode 실패 | 높음 | native host를 숨기지 않고 mode를 자동 해제하며 오류 표시 |
| 빠른 mode 전환/영상 전환의 stale async 결과 | 높음 | generation token 재검증 |
| overlay API가 다시 깨져도 mpv가 채택됨 | 높음 | load-time readiness handshake + fail-closed |
| draw playback에서 refresh 영구 연기 | 중간 | 마지막 요청을 보존하는 coalescing + source/runtime test |
| 기존 HTML5 경로 회귀 | 중간 | mpv 조건 안에서만 freeze/host 제어, fallback suites 재실행 |
