## 배경 및 목적

`DEVLOG/피드백-25-31-수정계획.md`를 기준으로 사용자 피드백 25~31의 7건을 구현했다. 계획서의 앵커와 before 블록을 실제 코드와 대조한 뒤, 작업 1 → 2 → 3 → 4 → 5 순서로 적용했다.

작업 4의 독립 검토에서 EOF와 빠른 재생/정지 전환 중 이전 스크린샷 요청이 최신 복귀 상태를 앞질러 검은 화면을 만들 수 있는 실제 경합을 발견했다. 이 계획서 범위의 안전성 보완은 사용자 승인 후 추가했고, 2026-07-13 최신 `main` 리베이스에서는 최신 공용 mpv review-freeze 경로에 같은 보호를 통합했다. 최신 성공 캡처만 도구 패널을 다시 보이게 한다.

## 작업 1 — 드로잉 모드 휠 줌

- 커밋: `3e0f838 feat: 드로잉 모드에서 마우스 휠 줌 허용 (피드백 30)`
- `app.js`의 비디오 휠 줌 가드가 드로잉 모드 자체를 막지 않도록 바꾸되, 실제 획을 그리는 동안에는 계속 막도록 했다.
- 회귀 테스트가 드로잉 모드에서의 허용과 획 작성 중 차단을 함께 확인한다.

## 작업 2 — `V` 선택 도구 단축키

- 커밋: `0e4c0b5 feat: 드로잉 모드 V 단축키를 선택 도구 활성화로 전환 (피드백 26)`
- 기존의 드로잉 모드 종료용 하드코딩 `V`를 제거하고, 설정 가능한 `drawingSelectTool` 단축키로 연결했다.
- 단축키 설정의 `그리기 보조` 항목과 도움말을 함께 갱신했고, `Shift+V`, `Ctrl+Alt+V`, 입력창 텍스트 입력 경로는 유지했다.

## 작업 3 — mpv 호스트 커서 경합

- 커밋: `4ccecd1 fix: mpv 호스트 창 mousemove 전달 제거로 커서 깜빡임 해소 (피드백 27·29·31)`
- mpv 임베드 호스트와 오버레이 호스트를 계속 클릭 관통으로 유지하되, `forward: true`를 제거해 네이티브 호스트가 mousemove마다 OS 커서를 다시 지정하지 않게 했다.
- 두 호스트 테스트는 `setIgnoreMouseEvents(true)` 호출을 확인하도록 갱신했다.

## 작업 4 — 드로잉 재생 freeze/패널 전환

- 커밋: `3a4c50b fix: 드로잉 모드 재생 시 mpv 영상이 보이도록 freeze/도구 패널 상태 전환 (피드백 25)`
- 사용자 승인 경합 보완은 최신 `main`의 공용 mpv review-freeze 관리 경로에 통합했다. 원래의 별도 보완 커밋은 리베이스 중 중복이어서 적용하지 않았고, 같은 보호는 이 최종 작업 4 커밋에 포함된다.
- 재생 중에는 드로잉 도구 패널을 `playback-hidden`으로 숨겨 mpv 호스트가 보이게 하고, 정지·EOF에서는 새 freeze 프레임이 준비된 뒤 패널을 복귀시킨다.
- 복귀 토큰과 freeze 토큰으로 오래된 비동기 스크린샷 결과를 무시한다. freeze 캡처가 실패하면 패널을 성급히 보이지 않아 검은 화면보다 실제 mpv 영상이 남도록 안전하게 처리한다.
- 재생 중 숨겨진 상태에서는 `frameUpdate`와 120ms mpv 상태 폴링의 `timeupdate`가 새 freeze 갱신을 예약하지 않도록 해, 정지 복귀와 경합하지 않게 했다. 정지 직후 시크로 캡처가 오래되어 취소되면 패널을 다시 보이고 새 freeze 갱신을 예약해 영구히 숨는 경로도 막았다. PR 독립 리뷰에서 확인한 보완이다.

## 작업 5 — 전체화면 컨트롤바

- 커밋: `dcca628 fix: 전체화면 컨트롤바를 레터박스 위에 미러 표시하고 축소를 스냅으로 안정화 (피드백 28)`
- `getMpvFullscreenControlsInset()`이 전체화면·컨트롤 표시·컨트롤 영역 측정 가드를 거친 뒤, 시크바 돌출 높이까지 포함한 최종 inset을 한 번 계산하도록 바꿨다.
- 중앙 고정 영상의 하단 레터박스 여백이 inset보다 크면 화면을 축소하지 않고 컨트롤바를 그 여백에 표시한다.
- DOM 컨트롤바를 mpv 네이티브 오버레이로 미러하고, 전체화면 컨트롤 표시 전환이 끝난 뒤 한 번 강제 동기화한다.

## 진행 중이던 보간 작업 분리

- 초기 선커밋: `be098e8 feat: mpv 재생 중 프레임 보간(진행 중 작업 커밋)`
- 계획서 §공통-6-1에 따라 기존 미커밋 rAF 보간 변경(`renderer/scripts/modules/video-player.js`, `scripts/tests/mpv-runtime-source.test.js`)을 작업 5보다 먼저 독립 커밋했다.

- 작업 5 커밋은 `app.js`와 그 작업의 source 단언만 포함하며, 보간 변경을 섞지 않았다.

## 계획서와 실제 코드의 불일치

계획서의 구현 전 확인과 before 블록은 모두 실제 코드와 일치했다. 작업 4의 추가 경합 보완은 불일치가 아니라 독립 검토에서 확인된 실제 안전성 문제에 대한 사용자 승인 범위 확장이다.
