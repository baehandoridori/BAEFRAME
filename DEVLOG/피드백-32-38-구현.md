## 배경 및 목적

BAEFRAME은 mpv 3중 네이티브 창(메인 DOM / MPVEmbedHost / MPVOverlayHost) 구조라 DOM이 mpv 위에 보이려면 **차단(숨김)** 또는 **미러링** 두 경로만 가능하다. 이번 묶음은 드로잉/재생 경험 최적화(32), mpv 가림 해소(사용자 지시·37), 드로잉 워크플로 편의(33·34·35), 버전 비교 UX(36A·36B)를 다룬다.

## Phase별(작업별) 구현 내용

### 작업 1 — 드로잉/재생 렉·"확대·축소 진동" 제거 (피드백 32)
- `drawing-manager`에 **paintStamp**(캔버스 픽셀이 실제 바뀔 때만 증가하는 카운터) 도입. 증가 지점 5곳: `_renderFrameSync` 렌더키 변화(`_notePlaybackRenderKey`), async `renderFrame` 완료, `_redrawKeyframeFromRecords`, drawend 구독·`_onSelectionCommitted`, `clearAll`/`reset`.
- `app.js`의 `getCompositedDrawingOverlayDataUrl`에 **paintStamp 캐시** — 재생 중 입력이 없으면 동일 PNG를 재인코딩하지 않음. 스트로크/선택 조작 중(`canvasBusy`)에는 캐시 무효화. 어니언 스킨이 꺼져 있으면 투명 PNG 인코딩 생략.
- `mpv-embed/overlay-host`의 `updateBounds`에 **동일 bounds early-return**(`_boundsEquals`) — 네이티브 창 계단식 리사이즈 방지. overlay는 z-order 보존을 위해 `moveTop`을 항상 호출. `setVisible(true)`·`destroy()`에서 캐시(`_lastAppliedScreenBounds`) 리셋.

### 작업 2 — mpv 가림 UI 오버레이 미러 (사용자 지시)
- `serializeMpvOverlayHtml`에 `compositionLayerPanel(.open)`·`videoZoomControls`·`videoCommentOverlayControls`를 미러 대상으로 추가(`.filter(Boolean)`로 null 방어). `MPV_MIRRORED_OVERLAY_SELECTOR`에 3종 셀렉터를 변이 트리거로 등록(작업 1 캐시가 프레임당 재인코딩을 막으므로 안전 — 그래서 작업 1 선행).

### 작업 6 — 버전 전환 시 위치 유지 (피드백 36A)
- `loadVideo`/`loadVideoWithMpvPilot`에 `initialTime` 옵션 + 모듈 헬퍼 `resolveInitialFrameFromOptions`(fps 확정 후 초→프레임, totalFrames 클램프). mpv는 `mpvInitialFrame`으로 초기 seek, HTML5는 reveal 지연 유무와 무관하게 위치 복원. 버전 전환 콜백이 `currentTime`을 `initialTime`으로 캡처. fps가 다른 버전 간에도 벽시계 시간 유지.

### 작업 7 — 이전 버전 댓글 읽기 전용 (피드백 36B)
- 댓글 탭 필터바에 "다른 버전 댓글 보기" 드롭다운. 선택 버전 `.bframe`를 `loadReview`로 읽어 마커 평탄화 후 `previousVersionComments`(순수 표시용 로컬 상태)에 저장. `updateCommentListImmediate`의 빈/일반 분기 모두에서 목록 하단에 고스트 섹션 렌더(`comment-ghost-*` 전용 클래스로 기존 `.comment-item` 바인딩과 분리 — 검증에서 지목된 이중 바인딩 블로커 회피). 클릭 시 초 기준 fps 재매핑 seek. 버전 전환·파일 로드 시 자동 해제. `commentManager` 오염 없음.

### 작업 8 — 획 단위 선택 (피드백 35)
- **래스터 리프트** 방식: `findStrokeAtPoint`(점 히트테스트, 최신 획 우선)로 클릭 획을 찾고, 키프레임에서 제거·재합성 후 그 획만 오프스크린에 그려 기존 floating 선택(`beginStrokeFloating`)으로 만든다. 이동·복사·붙여넣기·커밋·undo는 기존 픽셀 선택 기계 그대로 재사용. `_onStrokeSelectRequest`는 **읽기 전용 히트 판정을 먼저** 수행해 빈 곳 클릭 시 빈 키프레임 생성 부작용을 막고, redraw await 중 재진입 가드/실패 원복 처리.

## 계획서와 실제 코드가 달랐던 지점 (반드시 기록)

1. **[전역·중요] `npm run format` 미실행 — 스코프 eslint --fix로 대체.**
   계획서 공통규칙 6·사용자 규칙 5는 커밋 전 `npm run lint:fix && npm run format`을 지시하나, 이 저장소는 baseline(`4c8a2af`)에서 **prettier-clean이 아니다**. `npm run format`(`prettier --write .`)을 실행하면 건드리지 않은 파일 ~250개(`.prettierrc` 포함)가 리포맷되어 **절대 규칙(명시되지 않은 포맷 변경 금지·범위 한정)** 과 정면 충돌한다. 따라서 전역 prettier는 실행하지 않고, 각 작업의 **변경 파일에만 `eslint --fix`를 스코프 적용**했다(이 저장소 eslint는 prettier 포맷을 강제하지 않아 baseline lint가 미포맷 코드에서 0 error 통과). 신규 라인은 house style(2스페이스/싱글쿼트/세미콜론)로 수기 작성. `npm run lint`는 0 error 유지.

2. **[작업 2] 계획서 미언급 기존 단언 갱신** — `serializeMpvOverlayHtml`에 항목 추가 + `.filter(Boolean)` 도입으로, 계획서 (d)가 검토하지 않은 `mpv-runtime-source.test.js:1080`의 `/elements\.controlsBar\s*\]\.forEach/` 단언이 깨졌다. `elements.controlsBar` 뒤에 항목이 붙고 배열이 `].filter(Boolean).forEach`로 닫히므로, 해당 단언을 `/elements\.controlsBar,[\s\S]*?\]\.filter\(Boolean\)\.forEach/`로 갱신(같은 커밋).

3. **[작업 6] 계획서 미언급 기존 단언 갱신** — HTML5 seek 호출을 `seekInitialVideoFrameBeforeReveal(html5InitialFrame())`로 바꾸면서, 계획서 (d)가 mpv-runtime만 언급하고 놓친 `cutlist-runtime-source.test.js:207`의 `/await seekInitialVideoFrameBeforeReveal\(initialFrame\)/` 호출부 단언이 깨졌다. `html5InitialFrame()`로 갱신(같은 커밋). 함수 정의 단언(:205)은 파라미터명 불변이라 그대로 통과.

4. **[작업 8] 계획서 내부 불일치(구현 vs 테스트) — 테스트를 구현에 맞춤.**
   계획서 (c)② 구현은 `findStrokeAtPoint(readRecords, point)`(빈 키프레임 생성 부작용 방지용 "읽기 전용 우선" 블로커 수정)인데, (d)② 테스트는 `findStrokeAtPoint(records, point)`를 단언한다(`records`는 이후 편집용 변수). 구현이 substantive하므로 그대로 두고 테스트 단언을 `findStrokeAtPoint\(readRecords, point\)`로 수정.

5. **[경미] 라인 오프셋** — 작업 1 onionDataUrl 앵커가 계획서 표기 `:7654` vs 실제 `:7660`(+6). 계획서가 "±수십 줄 오차, 앵커로 재확정" 명시. 앵커 재확정 후 내용 일치. 그 외 모든 before 블록은 문자 단위 일치(사전 검증 8/8 `beforeBlocksMatch=true, discrepancies=0`).
