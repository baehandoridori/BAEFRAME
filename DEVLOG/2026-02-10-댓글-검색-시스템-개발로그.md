# 2026-02-10 댓글 검색 시스템 개발로그

## 배경
- 댓글 수가 늘어나면서 특정 작성자/문구/프레임 범위를 빠르게 찾기 어려웠다.
- 해결 여부(전체/미해결/해결됨) 필터와 함께 검색 흐름을 유지할 필요가 있었다.

## 목표
- 댓글/답글/작성자/프레임 범위를 단일 검색창에서 탐색할 수 있게 한다.
- 검색 결과에서 매칭 구간을 시각적으로 빠르게 식별할 수 있게 한다.
- 기존 댓글 패널 UI와 충돌 없이 접고/펼치는 검색 UX를 제공한다.

## 구현 내용
- 댓글 패널 상단 필터 영역에 검색 토글(돋보기) 버튼 추가.
- 검색창 기본 접힘 상태, 토글 클릭 시 확장/재클릭 시 닫힘 동작 구현.
- 검색어 입력 시 댓글/작성자/답글/프레임 텍스트를 통합 검색.
- 매칭 텍스트에 하이라이트 + 글로우 피드백 적용.
- 검색 상태에 따라 결과 카운트/목록 렌더 상태를 동기화.

## UX 보완
- 플레이스홀더를 한글로 통일해 사용 맥락을 즉시 이해하도록 조정.
- 예시 키워드가 순환 노출되도록 처리해 입력 가이드를 제공.
- Clear/Esc 등 검색 제어 동작을 패널 흐름에 맞춰 정리.

## 확인 항목
- 필터 탭과 검색창 레이아웃 충돌 없음.
- 검색어 입력/삭제 시 결과와 강조 표시가 즉시 반영됨.
- 검색창 재열기 시 이전 상태 유지 정책이 의도대로 동작함.
